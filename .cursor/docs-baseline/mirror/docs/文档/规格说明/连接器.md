# 规格说明·连接器：遗留系统适配器

| 版本 | v0.3.0 |
|------|--------|
| 范围 | 接口抽象 + Mock；**平台支持 MES/ERP 等 10 类**；**三条写入路径 Path A/B/C**（ADR-005） |
| 决策 | 2026-06-16：Path A 灯塔 ERP+钉钉；Path B 通用 MES 写；Legacy `system` 枚举 **Day1 定稿 10 类** |

## 1. 职责

- 统一读/写企业 Legacy 系统（见 §2 系统清单）
- **仅** `execution_service` 可调 `write`
- 其他模块仅 `read`

## 2. Legacy 系统清单（枚举定稿）

| system | 全称 | 典型域 | Phase 1 |
|--------|------|--------|---------|
| `mes` | Manufacturing Execution System | 报工、工序、现场执行 | **Path B 实现（L2 写）** |
| `erp` | Enterprise Resource Planning | 订单、计划、库存账务、报工落账 | **Path A 实现（L2 写+读）**；Path B L0 只读 |
| `wms` | Warehouse Management System | 仓储、发料、库位 | 预留 stub |
| `oa` | Office Automation | 审批、协同（含 IM 入口） | 预留 stub |
| `crm` | Customer Relationship Management | 客户、商机、销售订单 | 预留 stub |
| `csm` | Customer Service Management | 售后、服务工单、客诉 | 预留 stub |
| `aps` | Advanced Planning & Scheduling | 排产、产能、交期模拟 | 预留 stub |
| `plm` | Product Lifecycle Management | BOM、工艺、工程变更 | 预留 stub |
| `qms` | Quality Management System | 检验、不合格、质量追溯 | 预留 stub |
| `mock` | 内置 Mock | 底座测试 | Phase 1 默认 |

> **CSM** 指客户服务/售后域；若客户现场为 **SCM（供应链）**，仍走 `erp` / `wms` / `aps` 组合覆盖，不单独增加 `scm` 枚举，避免与 ERP 边界重叠。

### 2.1 Python 接口（ADR-001）

```python
from typing import Literal, Protocol
from pydantic import BaseModel

LegacySystem = Literal[
    "mes", "erp", "wms", "oa",
    "crm", "csm", "aps", "plm", "qms",
    "mock",
]

class Connector(Protocol):
    system: LegacySystem
    vendor: str

    async def health_check(self) -> "ConnectorHealth": ...
    async def read(self, op: "ReadOperation") -> "ReadResult": ...
    async def write(self, op: "WriteOperation") -> "WriteResult": ...
```

> 实现位于 `os_core/connector_sdk/`；**仅** `execution_service` 调用 `write`。

## 3. 接入路线图（锁定 · ADR-005）

### 3.0 三条写入路径（并存，按客户选型）

> 权威出处：[ADR-005](../架构/架构决策记录-005-双写入路径与灯塔定版.md)。L0 机制（Graph→Rule→Execution→Audit→Revert→对账）**三条路径完全相同**；差异仅在 Connector Pack 组合与 Data-L0 账本落点。

| 路径 ID | 商业 SKU | 典型客户 | 生产写落点 | 读路径 | Phase 1 |
|---------|----------|----------|------------|--------|---------|
| **Path A** | **Starter-A** | **哈森灯塔**（无独立 MES） | `conn-erp-{vendor}-write` L2 | ERP + 钉钉 | **灯塔主路径** |
| **Path B** | **Starter** | 有 MES 的离散制造厂 | `conn-mes-{vendor}` L2 写 | ERP L0 读 | 平台通用默认 |
| **Path C** | **Starter-B-Lite** | 无 ERP/MES（应急） | `conn-mes-builtin` | PG `builtin_ledger` | 预埋，Y2+ 按需 |

### 3.1 Path A · 哈森灯塔（2026 主路径 ★）

> 哈森 **无独立 MES**；**ERP 为主系统**，**OA = 钉钉**。

| 能力 | Pack / Registry | 级别 | Phase 1 |
|------|-----------------|------|---------|
| ERP 读（工单/库存） | `conn-erp-{vendor}-read` | L0 | ✅ |
| **ERP 写（报工+Revert）** | **`conn-erp-{vendor}-write`** | **L2** | **✅** |
| 钉钉入口/待办/确认 | `conn-dingtalk` | L3 通知 + H5 | ✅ |
| MES | — | — | **不接** |

**写路径**：`WORK_REPORT` → `execution_service` → **`conn-erp-*-write`**（非 `conn-mes-*`）。

**Gate 0' 样例**：见 [文档/连接器/erp/kingdee/samples/](../连接器/erp/kingdee/samples/)。

详见 [14-一年冲刺路线图](../../准备/2026-06-16/14-一年冲刺路线图与并行研发.md) §二。

### 3.2 Path B · 平台通用（有 MES 厂）

| 阶段 | 系统 | 能力 | 说明 |
|------|------|------|------|
| **Phase 1** | **MES** | L2 写 + Revert | **唯一生产写路径** |
| **Phase 1** | **ERP** | L0 只读 | 工单/订单/库存；对账 read-back |
| Phase 2 | ERP | L2 写（选配） | 回写、计件草案 |
| Phase 2 | OA / IM | L0 + L3 通知 | 钉钉/企微 |
| Phase 2 | WMS | L0 只读 | 缺料/发料/库位 |
| Phase 2 | **QMS** | L0 只读 | 检验结果、NCR 查询；轻量追溯 Graph |
| Phase 2 | **APS** | L0 只读 | 排产计划、产能负荷（建议只读，写回谨慎） |
| Phase 2 | **PLM** | L0 只读 | BOM/工艺版本只读引用 |
| Phase 2 | **CRM** | L0 只读 | 销售订单、交期承诺查询 |
| Phase 2 | **CSM** | L0 只读 | 客诉/服务单只读 |
| Phase 3 | OA / WMS | L2 写（选配） | 审批、发料确认 |
| Phase 3 | QMS | L2 写（选配） | 检验记录、Hold/Release（高敏感） |
| Phase 3 | CRM / CSM | L1–L2 | 订单状态回写、服务闭环（按客户 Pack） |
| Phase 3+ | APS / PLM | L1 建议为主 | 排产建议、ECN 通知；**避免** Agent 直接改 BOM/主排产 |

**Path B 原则**：**MES 写 + ERP 读**。Gate 0' 样例见 [mes/](../连接器/mes/) 目录。

### 3.4 扩展性约束（Coding 时必须遵守）

1. **`LegacySystem` 枚举已含 10 类**（§2），**禁止**为新增厂商新增 `system` 类型；新厂商用 `{system}-{vendor}` 注册。
2. **ConnectorRegistry** 未注册 → `CONNECTOR_NOT_CONFIGURED`（403），禁止崩溃或 silent no-op。
3. **License Pack** 按 `conn-{system}-*` 授权；未购 → 硬拒绝。
4. **DSL `connector_ops`** 只增映射条目，不改 `execution_service` 主流程。
5. **Graph** 扩展新系统 → **clone 新版本** + freeze，不改旧 frozen 图。
6. **禁止** Phase 1 业务代码出现 `if (system === 'qms')` 硬分支；统一 Registry + Pack。
7. **禁止** `if (customer == 'hasen')`；路径差异 **仅** 通过 tenant License + Pack 绑定（ADR-005 §6）。
