## 1. 职责

在 **frozen Graph** 与 **DSL Plan** 之间做 **默认拒绝（deny-default）** 授权判定：

```text
DslPlan.steps[].verb + actor + context
  → RuleSet（绑定 graph_id + graph_version）
  → effect: allow | deny
  → deny 则 Harness/Execution 不得继续
```

**禁止**：Agent、MCP、Studio 绕过 Rule Engine 直调 `execution_service`。

---

## 2. RuleSet 生命周期

| 状态 | 可编辑 | 可绑定执行 |
|------|--------|------------|
| `draft` | ✅ | ❌ |
| `frozen` | ❌ | ✅（须 graph 同版 frozen） |
| `deprecated` | ❌ | ❌（新执行拒绝） |

与 Graph 相同：**freeze 后不可变**；变更须 clone 新 RuleSet 版本并重新 freeze。

---

## 3. 规则模型

```typescript
interface Rule {
  id: string;                    // rule-{name}
  effect: 'allow' | 'deny';
  subjects: string[];            // role:worker, role:supervisor
  actions: string[];             // CMV 动词，如 WORK_REPORT
  conditions?: Condition[];
  priority: number;                // 高优先；deny 与 allow 冲突时 deny 胜（AC R-03）
}

interface Condition {
  field: string;                 // record.confirmed, record.age_hours, time.within_shift
  op: 'eq' | 'ne' | 'lt' | 'lte' | 'gt' | 'gte' | 'in' | 'not_in';
  value: unknown;
}
```

`default_effect` 默认 **`deny`**（ADR-002）。

---

## 4. 求值算法

```text
1. 加载 RuleSet(graph_id, graph_version)，status 须 frozen
2. 收集 candidate rules：actions 包含当前 verb
3. 按 priority 降序排序
4. 对每条 rule：subjects 匹配 actor.role；conditions 全部满足
5. 首个匹配的 deny → 403 RULE_DENIED
6. 首个匹配的 allow → allow
7. 无匹配 → default_effect（deny）
```

**Override**：Scope/Tenant Override **可 patch Rule 条件参数**（见 [覆盖配置](./覆盖配置.md)），**不得** patch `effect` 为 bypass Audit。

## 5. API（OpenAPI v1.1.1）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/v1/rulesets` | `?tenant_id=&graph_id=` 列表 |
| POST | `/v1/rulesets` | 创建 draft |
| GET | `/v1/rulesets/{rulesetId}` | 获取 |
| PUT | `/v1/rulesets/{rulesetId}` | 更新 draft |
| POST | `/v1/rulesets/{rulesetId}/freeze` | draft → frozen |
| POST | `/v1/rulesets/{rulesetId}/evaluate` | 预检（Harness 展示用） |

`RuleEvaluateRequest` / `RuleEvaluateResponse` 见 OpenAPI components。

---

## 6. 与 Graph / Execution 三元组

每次 L2 执行携带：

```text
(graph_id, graph_version, ruleset_id)
```

- `ruleset.graph_version` **MUST** 等于 `graph_version`  
- verb **MUST** ∈ `graph.allowed_dsl`（DSL Registry 二次校验，AC D-03）

## 7. 验收（AC-BASE-001）

| ID | 要点 |
|----|------|
| R-01 | 无匹配 rule → deny |
| R-02 | allow 通过 |
| R-03 | deny 优先于 allow |
| R-04 | graph 升级后旧 ruleset 拒绝或 422 |
| R-05 | frozen 不可 PUT |
