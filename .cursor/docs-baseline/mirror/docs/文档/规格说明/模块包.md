## 1. Pack 是什么

**Pack** 是 FactoryOS **可独立授权、版本化、可组合** 的最小商业与技术单元。三类 Pack 与 05 对齐：

| 类型 | ID 前缀 | 绑定运行时 |
|------|---------|------------|
| **Graph Pack** | `graph-` | frozen Graph 模板 + 节点/边约束 |
| **Capability Pack** | `cap-` | DSL 动作子集 + Agent 安全级别 |
| **Connector Pack** | `conn-` | Legacy `{system}-{vendor}` + 读写级别 |
| **Skill Pack** | `skill-` | AgentModule FSM + 绑定 Graph/Cap |
| **Override Pack** | `override-` | 三级 Override 配置集（见 覆盖配置规格） |
| **Implementation Pack** | `impl-package-` | export/import 快照（非运行时 Pack） |

未购买 Pack → API **403 `MODULE_NOT_LICENSED`**（ADR-002 R-10）。

## 2. Pack ID 命名规则

```text
{type}-{name}[-{variant}]

示例：
  graph-work-report
  cap-report-l2
  conn-erp-kingdee-write    # Path A 灯塔
  conn-mes-{vendor}         # Path B 通用
  conn-erp-kingdee-read
  conn-mes-builtin          # Path C B-Lite
  skill-work-report-v1
  override-scope-line-{line_id}   # 租户内实例，非全局 SKU
```

**规则**：

1. 全局 SKU 使用小写 kebab-case；**禁止**客户名出现在共享 Pack ID（vendor 仅在 Connector 层）
2. `{vendor}` 占位符在实施时替换为 Registry key（如 `conn-erp-kingdee-write`、`conn-mes-acme`）
3. 版本与 Pack ID 分离：Pack ID 稳定，版本见 §3

## 3. Pack 元数据 Schema

```typescript
interface PackDefinition {
  pack_id: string;                    // 全局唯一，如 "cap-report-l2"
  pack_type: PackType;
  display_name: string;
  description: string;

  version: string;                    // SemVer，如 "1.2.0"
  maturity: PackMaturity;             // M0–M5，见 §5

  requires: PackDependency[];         // 硬依赖，缺则拒绝加载
  recommends: PackDependency[];       // 软依赖，告警不阻断

  binds: PackBinding;                 // 运行时绑定

  license: PackLicenseMeta;           // 授权元数据

  docs?: {
    spec_ref?: string;                // 相对 文档/ 路径
    acceptance_ref?: string;
  };

  created_at: string;
  updated_at: string;
}

type PackType =
  | 'graph' | 'capability' | 'connector'
  | 'skill' | 'override' | 'implementation';

interface PackDependency {
  pack_id: string;
  min_version: string;                // SemVer 下限
}

interface PackBinding {
  graph_template?: string;            // graph-*: 模板路径
  dsl_actions?: string[];             // cap-*: 允许的 DSLAction.id
  agent_modules?: string[];           // skill-*: AgentModule.id
  connector_registry_key?: string;    // conn-*: "{system}-{vendor}"
  legacy_system?: LegacySystem;       // conn-*: 枚举
  max_agent_level?: 'L0' | 'L1' | 'L2' | 'L3';
}

interface PackLicenseMeta {
  revenue_layer: 'L0' | 'L1' | 'L2' | 'L3';
  sellable_standalone: boolean;
  default_plans: ('community' | 'starter' | 'pro' | 'enterprise')[];
}
```

**存储**（规划）：

- 全局定义：`文档/商业/packs/{pack_id}.json` 或 Phase 2 `pack-registry` 表
- 租户授权：`tenant_licenses` 表（Phase 2）

## 4. 租户 License Schema

```typescript
interface TenantLicense {
  tenant_id: string;
  plan: 'community' | 'starter' | 'pro' | 'enterprise';

  packs: LicensedPack[];

  scope: {
    factory_count: number;
    factory_ids?: string[];           // Enterprise 限定
  };

  im_connector?: string;              // conn-dingtalk | conn-wecom

  overrides?: string[];               // override 实例 ID 列表

  expires_at: string;                 // ISO8601
  issued_at: string;
}

interface LicensedPack {
  pack_id: string;
  version_constraint: string;         // 如 ">=1.0.0 <2.0.0"
  enabled: boolean;
}
```

**Phase 1 过渡**：单租户 JSON 文件或环境变量 `LICENSE_JSON`；接口与 Phase 2 相同。

## 5. Pack 成熟度 M0–M5（飞轮）

| 级别 | 名称 | 条件 | 可对外售卖 |
|------|------|------|------------|
| **M0** | Reserved | ID + 目录 stub | 否 |
| **M1** | Spec | SPEC + CAPABILITY-PACK-MAP 登记 | 否 |
| **M2** | Stub | Registry 返回 403；Mock 可测 | 否（可预售） |
| **M3** | Pilot | ≥1 厂 Shadow/UAT 通过 | 限量 |
| **M4** | GA | ≥2 厂生产 ≥30 天 + AC ≥5 条 | **是** |
| **M5** | Standard | ≥3 厂 + import 无需定制 | 主力 SKU |

D2 扩展 **入库共享 Pack** 须达到 **M3+**（08 §6.3）。

## 6. 校验点（运行时）

| 调用路径 | 校验 |
|----------|------|
| Agent 加载 Module | `capability_packs` + `skill` requires |
| Rule Engine allow | `cap-*` 含该 DSL |
| Execution Service | `cap-*` + `conn-*` + Graph frozen |
| Connector Registry | `conn-*` + registry key 存在 |
| Override 合并 | `override-*` 在 tenant License 内 |

**错误码**：

| 码 | 含义 |
|----|------|
| `403 MODULE_NOT_LICENSED` | Pack 未授权 |
| `403 CONNECTOR_NOT_CONFIGURED` | Connector 未注册（连接器规格） |
| `409 PACK_VERSION_MISMATCH` | 依赖版本不满足 |

## 7. Implementation Package（export/import）

D1/D2 结案必须 export（08 R4）：

```typescript
interface ImplementationPackage {
  package_id: string;                 // impl-package-d1 | impl-package-d2
  tenant_id: string;
  source_factory: string;
  exported_at: string;

  graph_versions: string[];             // frozen Graph checksum 列表
  rule_sets: string[];
  licensed_packs: LicensedPack[];
  overrides: OverrideDocument[];      // 见 覆盖配置规格
  connector_configs: ConnectorConfigSnapshot[];  // 无密钥，仅 endpoint/profile
  reconciliation_profile?: object;

  signature?: string;                   // 可选：实施方签字 hash
}
```

**禁止 export**：明文密钥、客户 PII  bulk；密钥走独立 Secret 通道。
