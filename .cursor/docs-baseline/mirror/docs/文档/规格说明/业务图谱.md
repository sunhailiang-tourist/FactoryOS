# 规格说明·业务图谱：冻结机制

| 版本 | v0.1.0 |
|------|--------|
| 范围 | 平台底座；**不含具体工厂业务节点** |
| Schema | [业务图谱.schema.json](../数据结构/业务图谱.schema.json) |

## 1. 职责

- 存储、版本化 Business Graph
-  enforce `draft → in_review → frozen` 状态机
- 提供 checksum 与 freeze 门禁
- 与 RuleSet 版本绑定

## 2. 状态机

```text
draft ──submit──► in_review ──freeze──► frozen ──deprecate──► deprecated
  │                    │
  └────edit (仅 draft/in_review)────┘

frozen / deprecated：禁止原地修改 nodes/edges/allowed_dsl
变更路径：clone → 新版本 draft → 再次 freeze
```

| 状态 | 允许读 | 允许 L0 查询 DSL | 允许 L2 写 DSL |
|------|--------|------------------|----------------|
| draft | ✅ | ✅（测试租户） | ❌ |
| in_review | ✅ | ✅ | ❌ |
| frozen | ✅ | ✅ | ✅ |
| deprecated | ✅ | ⚠️ 只读 | ❌ |

## 3. Freeze 策略（核心）

### 3.1 谁可以 freeze

- 角色：`role:graph_admin` 或 `role:implementation_lead`
- **禁止** Agent 自动 freeze（ADR-002 R-09）
- 底座阶段：通过 Admin API + 未来 web-admin UI

### 3.2 Freeze 前置条件

1. `status === in_review`（或允许 `draft` 直 freeze 的 `force` 仅 dev 环境）
2. `nodes.length >= 1`，图连通（start → … → end）
3. 每个 `action` 节点声明的 `dsl_verbs` ⊆ 全局 DSL 注册表
4. 已绑定 `RuleSet` 且 RuleSet.status === frozen
5. `checksum` 与 canonical 内容一致

### 3.3 Checksum 算法

```text
1. 取 graph 文档，剔除 metadata.frozen_at / frozen_by / updated_at
2. JSON canonicalize（键排序、无空白）
3. SHA-256 → 前缀 "sha256:"
```

### 3.4 Freeze API 行为

```http
POST /v1/graphs/{id}/versions/{version}/freeze
```

成功：

- `status` → `frozen`
- 写入 `metadata.frozen_at`, `metadata.frozen_by`
- 返回不可变 snapshot
- 写 Audit：`GRAPH_FREEZE`

失败：

- 409 若已 frozen
- 422 若校验不通过

### 3.5 执行门禁（`execution_service` 调用前）

```typescript
// 伪代码
if (graph.status !== 'frozen') throw GraphNotFrozenError;
if (graph.version !== request.graph_version) throw GraphVersionMismatchError;
if (verb not in graph.allowed_dsl) throw DslNotAllowedInGraphError;
```

## 4. 版本策略

| 变更类型 | 处理方式 |
|----------|----------|
| 文案 label | 新版本 patch：v1.0.0 → v1.0.1 |
| 增删节点/边 | 新版本 minor：v1.0.x → v1.1.0 |
| 语义变更（DSL 集合变化） | 新版本 major：v2.0.0 |

**禁止** 对 frozen 版本做 in-place 更新。

## 5. 底座模板

- 路径：[graphs/通病模板-草稿.json](../业务图谱/通病模板-草稿.json)
- 状态：**draft**，不代表任何工厂
- 用途：开发 `graph_service`、演示 D1 通病**结构**（查询→确认→写入→审计→可回滚）

工厂接入后：

```text
copy template → tenant workshop 填 entity_ref → in_review → frozen
→ 存 tenants/{tenant_id}/graphs/{id}/{version}.json
```

## 6. API 概要（详见 [OpenAPI v1.1.1](../接口/工厂操作系统-v1.1.yaml)）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /v1/graphs | 创建 draft |
| GET | /v1/graphs/{id} | 获取最新或 `?version=` 指定版本 |
| PUT | /v1/graphs/{id}/versions/{version} | 更新 draft/in_review |
| POST | /v1/graphs/{id}/versions/{version}/submit | → in_review |
| POST | /v1/graphs/{id}/versions/{version}/freeze | → frozen |
| POST | /v1/graphs/{id}/versions/{version}/clone | 新版本 draft |

## 7. 与 RuleSet 绑定

- RuleSet.graph_id + graph_version 必须指向 **frozen** Graph
- execution 时携带 `(graph_id, graph_version, ruleset_id)` 三元组
