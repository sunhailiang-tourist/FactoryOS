## 1. 定位：双极之一（终端智能极）

FactoryOS **双极架构**：

```text
终端智能极（本规格）     内核门禁极（Graph / Rule / DSL / Audit / Revert）
  极致简单、0 智、多模态          极致可靠、可审计、可回滚
           \                    /
            → Harness 汇合 → Execution → Legacy
```

**原则**：感知层可以 **尽可能聪明**；写 Legacy **永远** 经门禁，不因「识别置信度高」而跳过。

---

## 2. 职责边界

| 模块 | 做什么 | 禁止做什么 |
|------|--------|------------|
| **前端壳**（`h5-worker` / `web-admin`） | 采集语音/图/视频/扫码/点击；展示确认卡 | 直连 Connector；本地写 MES |
| **感知适配器**（`server/api` 路由层） | 上传、转码、限流、脱敏日志 | 业务规则判断 |
| **agent_orchestrator** | ASR/OCR/VLM 结果 → **结构化意图 + 填槽** → DSL **计划** | `connector.write`；自动 freeze Graph |
| **Harness** | 确认门、角色、渠道、dry_run 提示 | 修改 Rule 结果 |

---

## 3. 输入通道（Modality Registry）

| 通道 ID | 说明 | 主角色 | Phase | 必选 fallback |
|---------|------|--------|-------|---------------|
| `text` | 文字输入 / IM 内打字 | 全员 | P1 | — |
| `voice` | 语音 → ASR | 工人、主管 | P1 | `scan` 或 `tap` |
| `photo` | 单张/多张图 → OCR/VLM | 工人 | P1 | 手工选工单 + 数字键盘 |
| `video` | 短视频 → 关键帧 + VLM | 工人（计件/清点） | P2 | `photo` |
| `scan` | 工单/工序码 | 工人 | P1 | 语音报工单号 |
| `tap` | 大按钮、最近工单、常用量 | 工人 | P1 | — |
| `card_confirm` | 主管/老板确认卡片 | 主管、老板 | P1 | PC 列表 |

**车间铁律**：任何 L2 写操作，**至少两种通道** 可完成同一任务（防噪声/识别失败）。

---

## 4. 感知流水线

```text
1. Ingress      上传/流式 ASR；大小/时长限制；tenant + actor 绑定
2. Normalize    图像压缩、去 EXIF（可选）、音频降噪（可选）
3. Perceive     ASR / OCR / VLM（经 LiteLLM 或专用 Provider）
4. Structure    LangGraph FSM：意图匹配 Skill → Pydantic 填槽
5. Plan         输出 DSL 计划 JSON（未执行）
6. Harness      展示影响摘要 → 待确认（L2）或只读返回（L0）
7. Gate         Rule Engine → Execution（见执行规格）
```

**步骤 4–5 是唯一允许 LLM/VLM 的链路**（与 ADR-001 一致）。

## 5. 结构化输出契约

感知层 **不得** 向 Execution 传递自然语言；须产出 **PerceptionOutput**（[Schema](../数据结构/PerceptionOutput.schema.json)）。`agent_orchestrator` 合并 `tenant_id`、`graph_id`、`graph_version` 后产出 **DslPlan**（唯一可进 Harness 的对象）。

```json
{
  "skill_id": "work_report_v1",
  "intent_confidence": 0.92,
  "modality_used": ["voice", "scan"],
  "steps": [
    {
      "verb": "WORK_REPORT",
      "params": {
        "work_order_id": "WO-2026-001",
        "quantity": 50,
        "idempotency_key": "..."
      }
    }
  ],
  "clarifications": [],
  "perception_refs": {
    "asr_transcript": "...",
    "ocr_text": null,
    "media_object_ids": ["media-uuid"]
  }
}
```

| 字段 | 要求 |
|------|------|
| `intent_confidence` | 0–1；**&lt; 0.85** 时 MUST 进入澄清或 fallback UI |
| `steps` | 动词 ⊆ Skill.allowed_dsl ⊆ Graph.allowed_dsl（与 DslPlan.steps 同形） |
| `perception_refs` | 进 Audit；**不进** ExecutionRecord 写 Legacy 的 params |

**API 分层**：

| 端点 | 响应 |
|------|------|
| `POST /v1/perception/voice` · `.../image` | **PerceptionOutput** |
| `POST /v1/agent/plan` · MCP `tools/call` | **DslPlan** |
| `POST /v1/harness/confirm` | ExecutionRecord 或 dry_run DslPlan |

## 6. 多模态场景（与 Skill 绑定）

### 6.1 工人 · `skill-work-report-v1`

| 用户动作 | 感知 | 填槽 | 门禁 |
|----------|------|------|------|
| 「A123 完成 50」 | voice → ASR | 工单号、数量 | L2 + 主管确认（Rule） |
| 拍完成区堆垛 | photo → VLM/OCR | 数量建议 + 工单（扫码优先） | **必须** 用户点确认 |
| 扫工单码 + 点「50」 | scan + tap | 确定性高 | L2 确认 |

### 6.2 主管 · `skill-supervisor-shift`（D2）

| 用户动作 | 感知 | 输出 |
|----------|------|------|
| 「驳回线 3 最后三笔」 | voice/text | DSL 批量计划 → **逐条** 确认卡 |
| 待办列表点通过 | tap | 单条 WORK_REPORT 确认 |

### 6.3 老板 · `cap-analysis-l0`（Phase 2）

| 用户动作 | 感知 | 输出 |
|----------|------|------|
| 「今天哪条线最慢」 | voice/text | L0 QUERY / ANALYZE；**无写** |
| 日报卡片 | push | 预生成摘要（模板 + LLM 润色，数据来自 Connector 只读） |

## 7. 提供商与降级

| 能力 | Phase 1 | 降级 |
|------|---------|------|
| ASR | 钉钉/企微 JS-SDK 或云 ASR | 文字输入 |
| OCR | 通用 OCR API | 手工输入 |
| VLM 计数 | LiteLLM vision（可选） | 拍照后 **仅作建议**，人工改数 |
| LLM 填槽 | LiteLLM chat + structured output | 固定表单 |

**禁止**：识别失败时 **静默执行** L2 写操作。

---

## 8. 安全与隐私

- 原始音视频/图片：**Data-L1** 短期留存（可配置 TTL，默认 7–30 天）；**不进** Data-L3 数仓
- 人脸/身份证：**默认不采集**；若客户启用须单独 Pack + 签字
- 感知日志与 Audit 关联 `exec_id` / `plan_id`，支持事故回溯
- 多模态 **不得** 绕过 ADR-002 R-01～**R-11**

## 9. API 预留（Phase 1 后半）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/perception/voice` | 音频 → transcript + plan draft |
| POST | `/v1/perception/image` | 图片 → ocr/vision hints + plan draft |
| POST | `/v1/agent/plan` | 文本/结构化意图 → DSL plan |
| POST | `/v1/harness/confirm` | 确认后触发 Rule → Execute |

OpenAPI 路由见 [`工厂操作系统-v1.1.yaml`](../接口/工厂操作系统-v1.1.yaml)（`/v1/perception/*`、`/v1/harness/*`）；详细 request/response schema 随 `server/api` 骨架补全。
