## 1. 目标

Core 1.0 起 **全表预埋 `tenant_id`**，支持 Baseline 单租户与 Y1 多灯塔 **同一套代码、同一 deployable**，且 Gate 0 可测跨租户隔离。

---

## 2. Gate 0 隔离策略

| 项 | Core 1.0（Baseline～Y1） | S1 百级（ADR-007） |
|----|--------------------------|---------------------|
| **机制** | **应用层强制过滤** | **PostgreSQL RLS** 全表 + 应用层双保险 |
| **默认 tenant** | `'default'` | — |
| **注入** | `server/api` middleware 解析 `X-Tenant-Id` 或 JWT claim → `request.state.tenant_id` |
| **DB 查询** | Repository **必须** `WHERE tenant_id = :tid` | RLS policy 双保险 |

**决策 MT-01**：Gate 0 **不启用 RLS**（降低 W1 复杂度）；**S1 百级（≥20 tenant 或 ADR-007 触发）启用 RLS**。Path C B-Lite 启用 RLS 见 [产品B-Lite](./产品B-Lite无ERP内置账本.md) §builtin 表。

## 3. tenant_id 契约

### 3.1 HTTP

| 来源 | 优先级 |
|------|--------|
| `Authorization` JWT `tenant_id` claim | 1（Y2 OAuth） |
| Header `X-Tenant-Id` | 2（Y1 内部） |
| 单租户部署默认 | `'default'` |

**禁止**：请求 body 中的 `tenant_id` **覆盖** middleware 解析结果（body 字段仅用于校验一致，不一致 → 403）。

### 3.2 数据模型

所有 Data-L1 表 **MUST** 含：

```sql
tenant_id TEXT NOT NULL DEFAULT 'default'
```

索引：`(tenant_id, …)` 复合索引于 Graph、Execution、Audit 高频查询列。

### 3.3 跨租户

| 操作 | 期望 |
|------|------|
| 读/写他 tenant 的 exec_id | **403**（AC N-03） |
| Package import | 目标 tenant 由 API 路径或 body 指定；须 admin 角色 |
| MCP `/mcp/v1/{tenant_id}` | URL tenant **必须** 与 token claim 一致 |

---

## 4. scope_id（租户内二级域）

| 字段 | 用途 |
|------|------|
| `scope_id` | 产线 / 工位 / 设备；与 [覆盖配置](./覆盖配置.md) Scope Override 对齐 |
| 隔离级别 | **不**跨 tenant；同 tenant 内按 Rule + Override 限制 |

---

## 5. License 与 Pack

- `license_service` 判定 `(tenant_id, pack_id)` 授权  
- 未授权 Pack → **403 MODULE_NOT_LICENSED** + audit  
- Baseline stub：JSON 文件 `tenants/{id}/licenses.json` 或 env 全开
