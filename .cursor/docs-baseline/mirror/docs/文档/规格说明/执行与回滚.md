## 1. 职责

`execution_service` 是 **唯一** 可触发 Connector 写操作的模块。

```text
ExecuteRequest
  → 校验 Graph frozen + Rule allow + DSL registered + License
  → shadow / dry_run? → 模拟，不写 Legacy → status=simulated
  → before_snapshot（Connector read）
  → Connector write（非 shadow 时）
  → after_snapshot
  → ExecutionRecord + Audit + connector_trace
```

## 2. ExecuteRequest

```json
{
  "tenant_id": "string",
  "graph_id": "string",
  "graph_version": "string",
  "ruleset_id": "string",
  "verb": "WORK_REPORT",
  "params": {},
  "scope_id": "line-01",
  "actor": { "user_id": "u1", "role": "role:operator", "channel": "h5" },
  "dry_run": false,
  "idempotency_key": "wr-2026-06-16-001"
}
```

| 字段 | 说明 |
|------|------|
| `dry_run` | 单次请求模拟；**不写 Legacy** |
| `idempotency_key` | L2 写 **必填**；同 key 不重复写 Legacy |
| `scope_id` | 产线/设备/工位；与 Override Scope 对齐 |

**幂等语义（MT-02）**：见 §2.1。

## 2.1 幂等与去重（L2 写）

| 项 | 规则 |
|----|------|
| **适用范围** | 所有 CMV `level: L2` 动词 |
| **客户端** | `ExecuteRequest.idempotency_key` **必填**（min 8 字符，tenant 内唯一） |
| **服务端** | `(tenant_id, idempotency_key)` 唯一索引；命中则返回已有 ExecutionRecord，不二次写 Legacy |
| **Revert** | 新 `idempotency_key`；原 exec `status` → `reverted` |
| **Compensator params** | CMV 使用 `legacy_ref`；HTTP `POST /v1/execute/{exec_id}/revert` 由服务解析 |
| **TTL** | 幂等记录保留 ≥90 天 |

**租户级 Shadow**：`tenant.shadow_mode=true` 时，所有 L2 写自动模拟（见 §3.2）。

## 3. 状态流转

```text
pending → running → success
                 → failed
                 → simulated   ← shadow_mode 或 dry_run 完成
success → reverted（via compensator）
        → revert_failed
```

### 3.1 终态语义

| status | Legacy 是否变更 | 对账 Job 是否纳入 |
|--------|-----------------|-------------------|
| `success` | ✅ 已写 | ✅ |
| `simulated` | ❌ 未写 | ❌ |
| `failed` | ❌ | ❌ |
| `reverted` | 已补偿 | 原 success 不再纳入 |

### 3.2 Shadow / dry_run 规范（T-01 对齐）

| 触发 | ExecutionRecord | Legacy |
|------|-----------------|--------|
| `dry_run=true` | `shadow_mode=true`, `status=simulated` | 不变 |
| `tenant.shadow_mode=true` | `shadow_mode=true`, `status=simulated` | 不变 |
| 两者皆 false | `status=success`（或 failed） | 按路径写 ERP/MES/builtin |

- **禁止** shadow 时调用 Connector `write`（可调用 `read` 取 before_snapshot）
- `before_snapshot` / `after_snapshot` 在 simulated 时 **完整模拟**，供 UI 与对账演练
- Audit **正常 append**（含 simulated 执行）

## 4. ExecutionRecord（v0.2 要点）

除 v0.1 字段外，必知：

| 字段 | 说明 |
|------|------|
| `shadow_mode` | 是否影子/模拟执行 |
| `connector_trace[]` | 每次 Connector 调用：pack_id, verb, latency_ms, outcome |
| `scope_id` | Override 生效域 |
| `legacy_refs[]` | 写成功后 Legacy 引用，供 Revert / 对账 |

完整 Schema 见 [执行记录.schema.json](../数据结构/执行记录.schema.json)。

## 5. Revert 策略

1. 仅 `status === success` 且存在 `compensator_verb` 的记录可 revert
2. **`simulated` 不可 revert**（Legacy 未变更）
3. Revert 本身产生新 ExecutionRecord，`revert_of` 指向原 exec_id
4. 原记录 `status` → `reverted`
5. Rule 可限制：`record.age_hours <= 24` 等（租户 RuleSet 配置）
6. Compensator 执行同一 Executor 路径，不得直连 Connector

## 6. Audit

- 所有 Execute / Revert / Graph freeze 写 Audit 表（append-only）
- Audit 与 ExecutionRecord 通过 `exec_id` 关联
- **禁止** UPDATE/DELETE Audit 行（仅合规归档）

## 7. 对账 Job

```http
POST /v1/reconciliation/run
```

- 仅比对 `status=success` 且非 `reverted` 且非 `shadow_mode` 的记录
- 对每个 `legacy_refs` 调用 Connector **read-back**（按 Path A/B/C，见 [Shadow-Mode §2.3](./Shadow-Mode与对账规格.md)）
- 输出 drift 列表

```json
{
  "run_id": "uuid",
  "tenant_id": "string",
  "started_at": "datetime",
  "status": "ok | drift_detected",
  "drifts": [
    {
      "exec_id": "uuid",
      "field": "quantity",
      "expected": 10,
      "actual": 8
    }
  ]
}
```

Phase 1 W7–8 实现；底座 Phase 0 定义接口与结果 schema。

## 8. 错误码

| Code | HTTP | 说明 |
|------|------|------|
| GRAPH_NOT_FROZEN | 409 | Graph 未冻结 |
| RULE_DENIED | 403 | 规则拒绝 |
| DSL_UNKNOWN | 400 | 动词未注册 |
| DSL_NOT_IN_GRAPH | 403 | Graph 白名单不含 |
| REVERT_NOT_ALLOWED | 409 | 不可 revert（含 simulated） |
| CONNECTOR_ERROR | 502 | Legacy 失败 |
| MODULE_NOT_LICENSED | 403 | Pack 未授权 |
