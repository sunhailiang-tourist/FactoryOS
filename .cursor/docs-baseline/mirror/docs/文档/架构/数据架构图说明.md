## 一、四层数据模型（Data-L0～L3）

```text
┌─────────────────────────────────────────────────────────────────┐
│  Data-L3  分析/数据池（选配）                                      │
│  全量同步 · 看板 · 跨系统汇总 — 非 Overlay 默认 · 需书面采购        │
└───────────────────────────────▲─────────────────────────────────┘
                                │ 客户信任 + D1 结案后
┌───────────────────────────────┴─────────────────────────────────┐
│  Data-L2  业务缓存（按需 · TTL · 最小字段）                        │
│  工单快照 · 查询加速 · Graph 孪生引用 — 能读 ERP 就不落库           │
└───────────────────────────────▲─────────────────────────────────┘
                                │ Connector read / 写前快照
┌───────────────────────────────┴─────────────────────────────────┐
│  Data-L1  运行数据（必做 · RDS 生产/测试）                         │
│  Graph/Rule · Audit · ExecutionRecord · Evidence · 对账 · License │
└───────────────────────────────▲─────────────────────────────────┘
                                │ 唯一写路径成功后记录
┌───────────────────────────────┴─────────────────────────────────┐
│  Data-L0  客户账本（权威 · 不在 FactoryOS PG）                     │
│  ERP · MES · WMS · OA … — B-Lite 时 builtin 表模拟 L0            │
└─────────────────────────────────────────────────────────────────┘
```

## 二、数据存放位置（云资源映射）

| 数据类型 | 存放 | 环境 | 说明 |
|----------|------|------|------|
| Graph / Rule / Audit / Execution | **RDS** Data-L1 | 测试/生产 RDS | 全表 `tenant_id`；S1 起 RLS |
| 业务缓存 | **RDS** Data-L2 | 同上 | TTL 可配置 |
| 语音 / 图片 | **OSS** | 生产 | 30d TTL；**不进 PG** |
| 源码 / OpenAPI / Schema | **GitLab** | 测试域 ECS | 真源在 Git |
| ERP/MES 事实 | **客户系统** Data-L0 | 客户侧 | Connector 读写 |
| B-Lite 账本 | **RDS** builtin 表 | 生产 RDS | 模拟 Data-L0 |
| 规模预埋 | **RDS** 表 | W1 起 | `cell_id` · `connector_instances` · `outbox_events` · `tenant_quotas` |

## 三、写操作数据流

```text
工人/主管（钉钉 H5）
    │
    ▼
Harness 确认门 ──────────────────────────────┐
    │                                        │ 感知媒体
    ▼                                        ▼
Agent（LLM）→ DSL 计划 JSON              OSS（30d）
    │
    ▼
Rule allow/deny（无 LLM）
    │
    ▼
Execution Saga（无 LLM）
    │
    ├──► Connector write ──► Data-L0（ERP/MES/builtin）
    │
    └──► Audit + ExecutionRecord + Evidence ──► Data-L1（RDS）
              │
              └──► 对账 Job read-back ──► 核对 L0
```

**Revert**：读 Data-L1 before 快照 → Compensator → Connector 写回 L0。

## 四、按客户类型的数据策略

| 客户 | Data-L0 | Data-L1 | Data-L2 | Data-L3 |
|------|---------|---------|---------|---------|
| **A 有 ERP（哈森）** | 客户 ERP | 必做 | 按需最小 | 不默认 |
| **有 MES 厂** | ERP + MES | 必做 | 按需 | 不默认 |
| **C B-Lite** | builtin 表 | 必做 | 内置 | 可选另议 |

## 五、部署与规模（数据视角）

| 模式 | 数据隔离 | ADR-007 |
|------|----------|---------|
| **Pool** | 共享 RDS + RLS + tenant_id | S1 百级 |
| **Bridge** | 独立 PG schema | 合规 mid-market |
| **Silo** | 独立 RDS 实例 | Enterprise |

S0：单 Cell `cell-default`；S2：Cell Router `max_tenants=200`；S1 消息：Redis Streams + Outbox。

## 六、测试 vs 生产数据边界

| 原则 | 测试 RDS | 生产 RDS |
|------|----------|----------|
| 可重建 | ✓ 7 天备份 | ✗ 15 天备份 |
| 真实客户数据 | 脱敏/样本 | Shadow 起真实 |
| 混用 | **禁止** 测试连生产 | **禁止** |
