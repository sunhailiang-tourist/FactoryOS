# 架构决策记录 004：集成平台化（GIP）

| 字段 | 值 |
|------|-----|
| 状态 | **Accepted** |
| 日期 | 2026-06-16 |
| 范围 | Platform-L1 集成平台化；修正 MCP/Console/Edge 时间表 |
| 依赖 | ADR-001, ADR-002, ADR-003 |
| 战略详述 | [17-集成平台化战略(GIP)](../../准备/2026-06-16/17-集成平台化战略(GIP).md) |

## 背景

ADR-001 将 Platform-L1 定义为 `connector_sdk` + Registry，MCP 对外开放延至 Y3（ADR-000 T-11）。对标 Celonis（MCP Agent Tools GA）、Tulip（Connector + OPCH）、Boomi/StackOne（AI 生成 Connector YAML）后，**L0 信任内核设计领先，但 L1 集成工程化不足**，将导致：

- 每家 ERP 接入依赖 Python 定制，无法「集成只调用、短期不改 Core」  
- 私网 ERP 在 P1 阻塞  
- Package 飞轮无法 S1 小时级复制  
- AI 集成能力落后于 iPaaS，且无法与 L0 治理结合  

**GIP（Governed Integration Platform）** 作为 Platform-L1 的正式工程战略，**不修改 L0 写路径与 freeze 红线**。

## 决策

### 1. Platform-L1 = GIP 四层

| GIP 层 | 模块/目录 | Phase |
|--------|-----------|-------|
| **Runtime** | `os_core/connector_sdk/runtime/` | Core 1.0 |
| **MCP Gateway** | `os_core/mcp_gateway/` | Y1 末内部 GA；Y2 对外 |
| **Intelligence** | `integration/tools/connector-agent/` | P1 |
| **Studio** | `apps/web-admin`（`/studio/*`）；Y2 可拆 `apps/integration-studio/` | P1 MVP |

L0（graph/rule/execution/audit）**不在本 ADR 变更范围内**。

### 2. Connector Blueprint 为一等接入契约

- 规格：[Connector Blueprint 规格说明](../规格说明/Connector-Blueprint规格.md)  
- **80%** 厂商接入通过 `blueprint.yaml` + Runtime 解释执行  
- **20%** 复杂场景保留 Python `Connector` 类，须伴生 `blueprint.meta.yaml`  
- Blueprint **只映射 Canonical Manufacturing Verbs（CMV）**；禁止 vendor 私有动词进入 DSL Registry  

### 3. `integration/` 与 Core 物理分离

```text
os_core/       → Core v1.0 冻结（tag core-v1.0.0 后集成期不改）
integration/    → 所有 Connector/Graph/Skill Pack 与 tenant Override
```

- 新 ERP/新厂 **只改 `integration/`**  
- CI：`os_core/*` → AC-BASE-001 全量；`integration/*` → Pack contract tests  
- **禁止** `integration/` import `os_core/*` 非公开 API  

### 4. 三速接入（S1/S2/S3）

| 模式 | 周期目标 | 机制 |
|------|----------|------|
| S1 | ≤1 周 | import Implementation Package + Override |
| S2 | ≤2 周 | Connector Agent → blueprint.yaml → 人审 → Contract Test |
| S3 | 1～2 周 | Python Pack + meta blueprint |

所有模式必经：**Shadow → Contract Test → 对账 → 批准开写**。

### 5. MCP 时间表修正（废止 ADR-000 T-11 的 Y3-only 口径）

| 里程碑 | 内容 |
|--------|------|
| **Y1 末** | 内部 MCP Gateway GA：`tools/list` = 已授权 CMV；`tools/call` → DSL Plan JSON（**不直写 Legacy**） |
| **Y2** | 对外 MCP：OAuth 2.1 + tenant scope + 工具级 ACL + audit |
| **Y3** | Partner Pack 签名 + MCP Registry 目录化 |

MCP Gateway **必须** 经 `agent_orchestrator` → `execution_service` 写路径；禁止 MCP tool 直调 `connector.write`。

### 6. Edge Agent 提前（修正过渡架构 Y3-only）

- P1 交付 **Edge Agent PoC**（出站 mTLS/WSS；凭证 edge 侧解密）  
- 灯塔 ERP 在私网且无 Edge Agent → **D1 阻塞**  
- Y3 可选拆分为独立 `connector-host` 进程（过渡架构不变）  

### 7. Integration Studio 提前

- P1 MVP 六步：Connect · Discover · Map · Prove · Freeze · Export  
- 替代原「Y2～Y3 Implementation Console 才可用」的集成实施路径  
- 完整 Console（Override 可视化、Pack 市场）仍 Y2  

### 8. Shadow + 对账升为 Gate 0 P0

- `tenant.shadow_mode=true` 时 execution **只记不写** Legacy  
- 对账 Job K-01/K-02 从 P1 **升为 P0**（见 AC-BASE-001 修订）  
- Studio **Prove** 步与 Shadow 2 周标准对齐 [04-工厂实施手册](../../准备/2026-06-16/04-工厂实施手册.md)  

### 9. Connector 认证分级

- Bronze / Silver / Gold 定义见 [17 §5.6](../../准备/2026-06-16/17-集成平台化战略(GIP).md#56-认证分级)  
- 目录：`文档/连接器/catalog/`  
- 对外只承诺 **Silver+** Pack  

### 10. Gate 0 扩展验收（Core 1.0 完成标志）

在原有 AC-BASE-001 P0 基础上，**新增 P0**：

| 类别 | ID 范围 | 内容 |
|------|---------|------|
| Blueprint | B-01～B-04 | Runtime 解释 mock blueprint；mapping；revert/reconcile 声明 |
| Package | P-01～P-03 | export/import 跨 tenant；Override 差量 |
| GIP Trust | T-01～T-03 | shadow_mode；K-01/K-02 对账；tenant License 403 |
| MCP 内部 | M-01～M-02 | mock tool → DSL plan；不直写 Legacy |

**Core 1.0 tag 条件**：原 P0 + 上表全部 PASS。

## 不决策（留给后续 ADR）

- Edge Agent 具体协议版本号  
- MCP 对外 OAuth IdP 选型  
- Partner Pack 签名算法  
- SQL read Connector 是否 Core 1.0（建议 Y2）  

## 后果

### 正面

- L1 与 Celonis/Tulip/iPaaS **接入速度追平**，同时 **写治理全球领先**  
- Core 冻结可 enforce；集成团队有独立主战场  
- AI 能力可并行演进而不侵蚀 L0  

### 负面

- Core 1.0 范围扩大（Runtime + Pack Loader + 对账 P0）  
- 需新增 `integration/`、`mcp_gateway/`；Studio P1 在 `apps/web-admin`（`/studio/*`），Y2 可拆 `integration-studio/`  
- ADR-000 T-11、过渡架构 Console/MCP 时间表 **以本 ADR 为准**
