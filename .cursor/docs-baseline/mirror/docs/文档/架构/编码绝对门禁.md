## 0. 门禁宗旨

```text
读代码 = 读业务流程
写代码 = 写可审计的业务叙事（中文注释）+ 可验证的行为（测试）
```

**优先级**（与 ADR 一致）：可靠性 > 安全性 > 可追溯 > 可回滚 > 简洁。

## 2. 项目说明文档（每个「项目根」必备）

### 2.1 何谓「项目根」

| 路径 | 说明 |
|------|------|
| `FactoryOS/` | 仓库总根（`README.md`） |
| `os_core/` | 内核域根 |
| `os_core/<module>/` | 每个 Python 包根 |
| `apps/` | 应用域根 |
| `apps/api/` · `apps/web-admin/` · `apps/h5-worker/` · `apps/edge-agent/` | 各应用根 |
| `integration/` | 配置枢纽根 |
| `integration/tools/connector-agent/` | 工具子项目根 |
| `tests/` | 测试域根 |

### 2.2 README.md 必填结构（简明，建议每节 3～8 行）

```markdown
# <项目名>

## 是什么
（一句话 + 在 FactoryOS 中的位置）

## 主要功能
- 功能 1
- 功能 2

## 不负责什么
- 明确边界（避免膨胀）

## 上下游
- **上游**：谁调用本模块 / 谁依赖本目录
- **下游**：本模块调用谁 / 产出给谁

## 关键文件
| 文件 | 作用 |

## 相关文档
- 链接 ADR / 规格 / OpenAPI

## 本地开发
（仅应用根：如何启动、测试）
```

新建目录 **必须先有 README，再允许** 出现 `.py` / `.tsx` 实现文件。

## 3. 中文注释规范（强制）

### 3.1 总要求

| 项 | 要求 |
|----|------|
| 语言 | **中文**为主；专有名词可保留英文（FastAPI、CMV、tenant_id） |
| 覆盖率 | 每个源文件、每个对外函数/类、每个模型字段 |
| 目的 | 读者 **不查 ADR** 也能理解业务意图与调用链 |
| 禁止 | 无意义注释（`# 赋值`）、与被注释代码矛盾的注释、大段英文废话 |

### 3.2 文件头注释（每个 `.py` / `.ts` / `.tsx` 顶部）

**Python 模板**：

```python
"""
模块：<包路径或文件相对路径>
作用：<本文件在业务/架构中的职责，1～3 句>
业务关联：<对应 Platform 层 / Pack / Gate / 租户流程中的哪一段>
上游：<谁 import 本文件 / 谁 HTTP 调入>
下游：<本文件 import 谁 / 调谁 API / 写哪张表>
非职责：<本文件明确不做的事，可选>
关联文档：<相对路径 1～2 个>
"""
```

**TypeScript 模板**：

```typescript
/**
 * 模块：apps/web-admin/src/pages/studio/Connect.tsx
 * 作用：Integration Studio「连通」步 UI
 * 业务关联：Playbook G-CONNECT · 配置枢纽 system_relations
 * 上游：用户/实施顾问 · 路由 /studio/connect
 * 下游：POST /v1/integration/connect/test
 * 关联文档：文档/规格说明/Integration-Studio规格.md
 */
```

### 3.3 函数 / 方法注释（必有）

```python
def evaluate_ruleset(tenant_id: str, plan: DslPlan) -> RuleDecision:
    """
    对 DSL 执行计划做 Rule 授权判定（默认 deny）。

    业务：工人/Agent 提出的写库意图是否允许进入 Execution。
    上游：apps/api routers/rulesets · agent_orchestrator 产出的 DslPlan
    下游：execution_service.execute（仅 ALLOW 时）· audit_service 记审计

    参数:
        tenant_id: 租户 ID，多厂隔离键；来自 middleware 注入
        plan: DSL 计划 JSON，须已对齐 DslPlan.schema.json

    返回:
        RuleDecision：含 allow/deny 与命中规则 ID

    异常:
        RuleSetNotFound：租户未绑定 RuleSet
    """
```

### 3.4 字段级注释（模型 / Schema / 表结构）

**Pydantic / dataclass / SQLAlchemy 每个字段**：

```python
class ExecutionRecord(BaseModel):
    """单次受控写操作的运行记录（Data-L1 真源之一）。"""

    tenant_id: str
    """租户 ID。业务：多厂隔离；关联 tenants 表、全链路 Audit。"""

    shadow_mode: bool
    """是否影子执行。业务：True 时不调 Connector.write；关联 G-PROVE Gate。"""

    legacy_ref: str | None
    """Legacy 侧回执 ID。业务：对账 match_key；关联 Reconciliation Job。"""
```

### 3.5 复杂逻辑块

超过 **10 行** 或非显而易见的分支，必须有 **块注释** 说明业务分支含义（例如 Shadow、Revert、idempotency 命中）。

## 4. 代码质量：干净 · 精简 · 无冗余

### 4.1 原则

| 原则 | 说明 |
|------|------|
| **DRY** | 相同逻辑 **≥2 处** 必须抽取；禁止复制粘贴改两行 |
| **SRP** | 一个函数只做一层业务步骤 |
| **无重复功能** | 同一能力不得在 `apps/api` 与 `os_core` 各实现一遍 |
| **薄 API** | `apps/api` 仅路由/注入/校验；业务只在 `os_core` |
| **可删性** | 删文件后测试应明确失败，禁止「死代码」 |

### 4.2 冗余检查机制

| 层级 | 工具 / 动作 | 时机 |
|------|-------------|------|
| 模块边界 | `scripts/check_import_boundaries.py` | CI · `os_core/**` 变更 |
| 契约重复 | `check_openapi_schema_refs` · `check_cmv_sync` | CI |
| 复制粘贴 | `scripts/check_code_redundancy.py`（jscpd/ruff 封装） | CI · W1 起启用 |
| 人工 | PR Review：是否可并入 `shared_contracts` / 已有 api.py | 每次 PR |

**合并拒绝条件**：

- 重复实现已有 `os_core.<module>.api` 能力  
- 同一 `tenant_id` 分支硬编码（须 Override / integration）  
- 新增文件无文件头注释或无 README 更新  

### 4.3 允许的内联重复

- 测试里 **Arrange** 样板（但须用 fixture 收敛）  
- 类型字面量 **1 处** 的简单映射

## 5. 与架构红线的关系

本门禁 **不替代** ADR-002；编码时仍须遵守：

- 写路径仅 `execution_service`  
- Agent 无写 Legacy 权限  
- integration 不调 os_core 非公开 API  

注释须 **如实反映** 红线，不得注释写「可绕过 Rule」。
