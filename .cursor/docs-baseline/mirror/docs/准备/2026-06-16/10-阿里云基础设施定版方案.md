# 阿里云基础设施定版方案

> 版本：v1.0.0 | 日期：2026-06-16 | 状态：**已定版（采购依据）**  
> 决策：采用 **阿里云**；基础设施 **类型一次齐套**；Y1 规格 **够用有余量**；以后 **只升配、不新增组件类型**  
> 关联：[06-数据边界与部署策略](./06-数据边界与部署策略.md)、[03-锁定实施策略](./03-锁定实施策略.md)、[架构决策记录 001](../../文档/架构/架构决策记录-001-系统架构总览.md)

---

## 一、设计原则

| 原则 | 做法 |
|------|------|
| **基础设施完整** | 计算、数据库、缓存、对象存储、负载均衡、网络隔离 **Day1 齐套** |
| **复杂度也是成本** | 不把 PostgreSQL 塞进生产 ECS；不在「以后再加 RDS / SLB / Redis」 |
| **规格够用有余量** | 中等规格 + 磁盘/内存可扩容；避免 2C8G 顶满后被迫改架构 |
| **以后只升配** | ECS / RDS / Redis **升 CPU、内存、磁盘**；SLB 后 **加同规格 ECS**；不改 DNS、不新增一类托管服务 |
| **研发与生产分离** | GitLab 仅在测试域；生产域 **只跑 FactoryOS 运行时** |
| **接口文档不进运行时** | OpenAPI 以 Git 为真源；测试域 **Redoc 静态页** 展示；**不部署 YApi**（少 Mongo 一类运维） |

**一句话**：

> 拓扑一次定全，规格留余量；以后只纵向扩容，不「先凑合同机 PG，以后再拆 RDS」。

## 二、目标拓扑

```text
                         ┌─ 办公网 / VPN ─┐
                         │   Git push     │
                         ▼                │
              ┌──────────────────────────────┐
              │  测试 ECS（研发 + 验证域）       │
              │  GitLab CE + Runner            │
              │  FactoryOS staging           │
              └──────────────┬───────────────┘
                             │ 仅 VPC 内网
              ┌──────────────▼───────────────┐
              │  RDS 测试  PostgreSQL 16       │
              └──────────────────────────────┘

  钉钉 / 用户 HTTPS
         │
         ▼
┌─────────────────┐     ┌─────────────────────────┐
│  SLB（公网 HTTPS）│────►│  生产 ECS                │
└─────────────────┘     │  Nginx + FactoryOS API   │
                        └──────┬──────────┬─────────┘
                               │          │
                    ┌──────────▼──┐   ┌───▼────────┐
                    │ RDS 生产 PG16 │   │ Redis 1GB  │
                    └──────────────┘   └────────────┘
                               │
                    ┌──────────▼──────────┐
                    │ OSS（多模态媒体）     │
                    └─────────────────────┘
                               │ 出站（VPN / 专线 / 客户白名单）
                    ┌──────────▼──────────┐
                    │ 企业 A / B  MES · ERP  │
                    └─────────────────────┘
```

**租户模型**：两家企业 = 同一套生产栈上的 **两个 `tenant_id`**（逻辑隔离），非两套物理集群。

## 三、采购清单（Y1 定版规格）

### 3.1 网络（Day1 固定）

| 资源 | 配置 | 说明 |
|------|------|------|
| **VPC** | 1 个，例 `172.16.0.0/16` | 全环境唯一 VPC |
| **vSwitch** | 2 个 | `vswitch-test`、`vswitch-prod`；建议分可用区 |
| **安全组** | 2 套 | 见 §五 |
| **域名** | 1 个主域 + 子域 | `api.*`（生产 SLB）、`gitlab.*`（测试 EIP）；采购见 `12` §四 |
| **SSL** | **通配符 DV 证书 1 张**（定版采购） | 挂 SLB + 测试 ECS；免费 DV 仅作内测备选；钉钉 H5 需 HTTPS |

### 3.2 测试环境（研发 + UAT + CI）

| 资源 | 规格 | 说明 |
|------|------|------|
| **ECS 测试** | **ecs.g8i.xlarge**（**4 vCPU / 16 GiB**） | GitLab + Runner + FactoryOS staging |
| 系统盘 | 40 GB ESSD | OS |
| 数据盘 | **150 GB ESSD PL0** | Git 仓库、CI 产物、日志 |
| 带宽 | 5 Mbps 固定或按量 | push / CI |
| 公网 | **EIP**（或 VPN 入内、无公网更安全） | 研发访问 GitLab |
| **RDS 测试** | **PostgreSQL 16**，**2 vCPU / 4 GiB**，**50 GB**，自动备份 **7 天** | staging / UAT；连接方式与生产一致 |

### 3.3 生产环境（两企业 tenant）

| 资源 | 规格 | 说明 |
|------|------|------|
| **SLB** | 应用型负载均衡，HTTPS | 公网入口；健康检查；日后扩 ECS 不改域名 |
| **ECS 生产** | **ecs.g8i.xlarge**（**4 vCPU / 16 GiB**） | **仅** Nginx + Uvicorn（**4 workers**） |
| 系统盘 | 40 GB ESSD | OS |
| 数据盘 | **80 GB ESSD** | 应用日志、临时文件（**业务数据不在此**） |
| **RDS 生产** | **PostgreSQL 16**，**4 vCPU / 8 GiB**，**100 GB**，**存储自动扩容**，自动备份 **15 天** | Layer 1：Audit、ExecutionRecord、Graph/Rule |
| **Redis（Tair）** | **1 GB** 标准主从版 | 会话 / 限流 / 缓存；Phase 2 对账 Job 队列接口先接通 |
| **OSS** | 标准存储 **100 GB** + **30 天生命周期** | 语音 / 图片（感知层不进 PG） |

### 3.4 Day1 明确不引入

| 不引入 | 原因 |
|--------|------|
| Kubernetes / ACK | ADR-001：Phase 1 Modular Monolith |
| Kafka / 消息集群 | ADR-001：Phase 1 禁止分布式复杂度 |
| 生产上 GitLab / YApi | 研发设施不进生产域 |
| 生产 ECS 上自建 PostgreSQL | 与 RDS 职责重复；后期拆库 = 新建一类基础设施 |
| 第二套临时 PG | 测试 / 生产 **各一条 RDS**，拓扑对称 |

## 四、各节点软件清单

### 4.1 测试 ECS

| 组件 | 用途 | 备注 |
|------|------|------|
| GitLab CE（Omnibus） | 代码仓库、MR、CI 配置 | 见 §4.3 精简参数 |
| GitLab Runner | `concurrent=1～2` | Python 项目构建、部署 staging |
| FactoryOS staging | 与生产同镜像 / 同 Compose 结构 | 对接 RDS 测试 |
| Nginx | Redoc 静态 API 文档（可选） | 源：`文档/接口/工厂操作系统-v1.1.yaml` |

**测试域四类部署（用途清单）**——详见 [15-研发资源与云环境用途清单](./15-研发资源与云环境用途清单.md) §三：

| # | 部署类型 | 跑在哪 | 回答的问题 |
|---|----------|--------|------------|
| 1 | **代码仓库** | 测试 ECS · GitLab | 代码放哪、谁审 MR |
| 2 | **CI / 测试系统** | Runner + staging | push 后能否自动测、能否预发布 |
| 3 | **接口文档** | Nginx + Redoc | API 契约对内怎么查（**非 YApi**） |
| 4 | **验收库** | RDS 测试 | 迁移与联调数据（可重建） |

### 4.2 生产 ECS

| 组件 | 用途 | 备注 |
|------|------|------|
| Nginx | 反向代理、TLS 终结（若 SLB 已终结可仅 HTTP 内网） | |
| FactoryOS API | FastAPI + Uvicorn × **4** | ADR-001：workers ≈ CPU 核数 |
| （无 GitLab / 无 PG / 无 Redis 进程） | | Redis、PG 走托管 |

**生产域仅运行时**：源码、CI、接口管理（Redoc）**不进生产 ECS**；工人/主管经 SLB → API → RDS/OSS/Connector 出站。

### 4.3 GitLab 内存精简（4C16G 测试机建议）

在 `/etc/gitlab/gitlab.rb`：

```ruby
prometheus_monitoring['enable'] = false
grafana['enable'] = false
gitlab_rails['registry_enabled'] = false

puma['worker_processes'] = 2
sidekiq['max_concurrency'] = 10
```

Runner：`concurrent = 1`（CI 高峰时可临时调到 2）。

### 4.4 托管服务连接

| 服务 | 生产 ECS 环境变量（示例） | 网络 |
|------|---------------------------|------|
| RDS 生产 | `DATABASE_URL` → RDS 内网地址 | 安全组：仅生产 ECS → RDS 5432 |
| Redis | `REDIS_URL` → Tair 内网地址 | 安全组：仅生产 ECS → Redis 6379 |
| OSS | `OSS_ENDPOINT` / `OSS_BUCKET` | AK 走 RAM 角色或环境变量，不进 Git |

## 五、安全组与端口

### 5.1 测试安全组 `sg-test`

| 方向 | 端口 | 源 / 目标 | 说明 |
|------|------|-----------|------|
| 入站 | 443 | 办公网 IP 白名单 | GitLab HTTPS |
| 入站 | 22 | 堡垒机 / 办公 IP | SSH（建议密钥） |
| 出站 | 443 | 0.0.0.0/0 | 包更新、Webhook |
| 出站 | 5432 | RDS 测试安全组 | staging → PG |

### 5.2 生产安全组 `sg-prod-ecs`

| 方向 | 端口 | 源 / 目标 | 说明 |
|------|------|-----------|------|
| 入站 | 80 / 443 | SLB 后端服务器组 | 仅 SLB 转发 |
| 出站 | 5432 | RDS 生产安全组 | App → PG |
| 出站 | 6379 | Redis 安全组 | App → Redis |
| 出站 | 443 | 0.0.0.0/0 + 客户 MES/ERP 网段 | Connector、钉钉、LLM API |
| 出站 | OSS 内网 Endpoint | OSS | 媒体上传 |

### 5.3 RDS / Redis 安全组

| 资源 | 入站 |
|------|------|
| RDS 测试 | 5432 ← **仅** `sg-test` |
| RDS 生产 | 5432 ← **仅** `sg-prod-ecs` |
| Redis | 6379 ← **仅** `sg-prod-ecs` |

**禁止**：RDS / Redis 对公网开放。

## 六、与 FactoryOS 架构对齐

| 文档约束 | 本方案 |
|----------|--------|
| Modular Monolith，单 `apps/api` | 生产 **一台 ECS** 跑 API |
| PostgreSQL 16 | **RDS ×2**（测试 / 生产各一） |
| Phase 2 Redis + ARQ | **Redis Day1 就绪** |
| Layer 1 审计长期留存 | RDS 自动备份 + 可升盘 |
| 多模态媒体不进 PG | **OSS + 30 天 TTL** |
| 两厂 `tenant_id` 隔离 | 同一生产栈，逻辑隔离 |
| Uvicorn workers ≈ CPU 核数 | 4C 生产 → 4 workers |
| 数据权威在 ERP/MES | FactoryOS 仅存 Layer 1 + 必要 Layer 2 |

## 七、部署与发布路径

```text
开发 push → GitLab（测试 ECS）
         → Runner：pytest / contract test（AC-BASE-001 / UX-001）
         → 构建镜像或 rsync 到 staging
         → staging 验收（RDS 测试）
         → 人工或 tag 触发部署生产
         → SLB 后端滚动 / 单实例替换
         → 生产 RDS（迁移脚本经 CI gate）
```

**原则**：测试、生产 **同一 Docker Compose / 同一环境变量结构**，仅连接串与域名不同。

## 八、备份与恢复

| 数据 | 策略 |
|------|------|
| RDS 生产 | 云厂商自动备份 15 天 + 重大发布前手动快照 |
| RDS 测试 | 自动备份 7 天；可重建 |
| OSS | 跨区复制 **Y2 可选**；Y1 生命周期 30 天删除媒体 |
| GitLab | 内置备份 cron → 定期 dump 至 OSS |
| 恢复演练 | **每季度 1 次** staging 从 RDS 快照恢复验证 |

## 九、扩容路径（只升配，不改拓扑）

| 触发信号 | 动作 |
|----------|------|
| GitLab CI 慢 / OOM | 测试 ECS → **8C16G 或 8C32G** |
| API P99 延迟升高、CPU >70%  sustained | 生产 ECS → **8C16G**；或 SLB 后 **加第 2 台同规格 ECS** |
| Audit 表 >50GB / 慢查询 | RDS 生产 → **升盘** 或 **4C16G** |
| 对账 Job 常驻、队列积压 | Redis → **2G** |
| 第三家 tenant | 仍同一栈；优先 **RDS 升配** |
| 媒体量增大 | OSS 升配额；生命周期策略不变 |

**禁止**：为扩容而引入 ACK、自建 PG、把 GitLab 迁到生产 ECS。

## 十、成本粗估（华东，人民币）

| 资源 | 约月费（元） |
|------|-------------|
| ECS 测试 4C16G + 150G 盘 | 900～1,200 |
| ECS 生产 4C16G + 80G 盘 | 900～1,200 |
| RDS 测试 2C4G | 500～800 |
| RDS 生产 4C8G | 1,000～1,500 |
| Redis 1G 主从 | 200～400 |
| SLB + 流量 | 100～300 |
| OSS + 出网 | 100～200 |
| **合计** | **约 3,700～4,600 / 月** |

包年（约 6～7 折）：**约 2.5～3.5 万 / 年**起。  
3 年云 TCO 粗算：**约 13～17 万**（不含运维人力）。

**非云成本**（Cursor、研发期 LLM / 多模态 API、域名等）见 [12-软硬件与API成本框架](./12-软硬件与API成本框架.md)。  
**云资源用途细分**（测试四类部署 / 生产运行时 / 全流程对照）见 [15-研发资源与云环境用途清单](./15-研发资源与云环境用途清单.md)。  
Y1 **软硬件总账**（**已含云** + Cursor + 研发 API + 域名/SSL，不含人力）：约 **5.4～9.1 万 / 年**。

## 十一、采购检查清单

```text
□ VPC + test/prod vSwitch + 安全组（§五）
□ ECS 测试 4C16G + 150G 数据盘 + EIP（或 VPN）
□ ECS 生产 4C16G + 80G 数据盘
□ RDS 测试 PG16 2C4G 50G
□ RDS 生产 PG16 4C8G 100G（自动扩容）
□ Redis 1G 主从
□ SLB + HTTPS 证书
□ OSS bucket + 30 天生命周期
□ RAM 子账号 / ECS 实例角色（OSS、日志）
□ 域名解析：api.* → SLB；gitlab.* → 测试 EIP
□ MES/ERP 出站：VPN 或客户白名单（两家各自网段）
```

## 十二、与自建方案关系

本文档 **定版采用阿里云**。若未来客户要求机房私有化，应以 **同等拓扑**（测试 / 生产 / 库分离）映射为物理机，而非回退到「PG 与 App 同机」的极简形态。参见沟通记录中的自建 TCO 对比（不另文展开）。
