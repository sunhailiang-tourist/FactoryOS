## 一、业务链路模块化：三层拆分

FactoryOS 按 **Pack-L1～L3 三层可插拔模块** 设计与售卖：

```text
Pack-L1  流程模块（Graph Pack）       → 卖「业务场景」
Pack-L2  能力模块（Capability Pack） → 卖「能做什么」
Pack-L3  连接模块（Connector Pack）   → 卖「接什么系统」
```

### 运行时关系

```text
租户 License：graph-xxx + cap-xxx + conn-xxx
  ↓
用户入口（钉钉/企微/PC）
  ↓
Agent 只加载已授权 Module
  ↓
DSL 只调用已授权动作
  ↓
Connector 只访问已授权系统
```

未购买模块 → API **硬拒绝**（不仅隐藏 UI）。

## 二、Pack-L1：Graph Pack（流程模块）

| 模块 ID | 名称 | 包含链路 | 典型用户 |
|---------|------|----------|----------|
| `graph-work-report` | 生产报工 | 工单→报工→主管确认→Legacy 落账 | 工人、主管 |
| `graph-production-query` | 生产查询 | 产量/效率/异常（只读） | 主管、老板 |
| `graph-inventory-read` | 库存查询 | ERP 库存/缺料（只读） | 计划、仓库 |
| `graph-order-trace` | 订单追溯 | 订单→工单→延期根因 | 老板、计划 |
| `graph-purchase-l1` | 采购建议 | 建议采购，不执行 | 采购、财务 |
| `graph-purchase-l2` | 采购执行 | 建单+确认+回滚 | 财务（后期） |
| `graph-oa-notify` | 审批通知 | 只读审批+IM 推送 | 全员 |

**版本**：每个 Graph Pack 独立版本，与客户 frozen Graph 实例绑定。

## 三、Layer 2：Capability Pack（能力模块）

| 能力包 ID | DSL 示例 | Agent | 安全级别 |
|-----------|----------|-------|----------|
| `cap-read-l0` | `QUERY_*` | 查询 Agent | L0 只读 |
| `cap-report-l2` | `WORK_REPORT`, `WORK_REPORT_REVERT` | 报工 Agent | L2 确认后写 |
| `cap-analysis-l0` | `ANALYZE_*` | 分析 Agent | L0 |
| `cap-approval-notify` | `NOTIFY_*` | 通知 Agent | L3 |
| `cap-purchase-l2` | `PURCHASE_*` | 采购 Agent | L2（收费） |

### Agent 安全分级（与能力包绑定）

| 级别 | 模式 | 示例 | 商业化 |
|------|------|------|--------|
| L0 | 只读 | 查库存、订单 | 可免费 |
| L1 | 建议 | 建议采购，不执行 | 可免费/低价 |
| L2 | 执行+确认 | 报工、建采购单 | **收费** |
| L3 | 自动执行 | 日报、通知 | 低风险可含在套餐 |

## 六、商业化：免费 vs 收费

### 原则

```text
免费：占入口 + 只读价值（让人用起来）
收费：写操作、Connector、多厂、合规、私有化（让业务跑通）
```

### 模块级定价方向

| 类型 | 免费 | 收费 |
|------|------|------|
| Graph Pack（只读） | 1–2 个基础包 | 订单追溯、采购建议等 |
| Graph Pack（写） | — | 报工、采购执行等 |
| DSL 写 + Revert | — | 按动作包 |
| Connector 写 / 第二系统 | — | 按厂商+接口 |
| 多工厂 tenant | — | 按厂 |
| Implementation Studio | — | GIP 实施工具（ADR-004） |
| 审计长期留存 / 合规导出 | 基础 | 完整版 |

### 收入结构（前期现实）

```text
订阅/年费（模块 License）
+ 实施费（Connector + Graph freeze）
+ 定制费（特殊规则/报表）
+ 私有化部署与维护
```

避免「Agent 全免费 + Connector 天价」或「全免费无法覆盖实施」两种极端。

## 七、各端用户入口

### 7.1 角色 × 场景

| 角色 | 主要场景 | 频率 | 环境 | 交互 |
|------|----------|------|------|------|
| 工人 | 报工、查工单 | 高 | 车间，噪声 | 语音/拍照/大按钮 |
| 班组长/主管 | 确认报工、看异常 | 高 | 车间+办公室 | 手机 |
| 计划/仓库 | 查库存、缺料 | 中 | 办公室 | PC+手机 |
| 财务/行政 | 确认落库、审批 | 中 | 办公室 | **PC 为主** |
| 老板 | 看板、延期根因 | 低 | 随时 | 手机摘要+PC 细节 |
| IT/实施 | Graph freeze、规则 | 低 | 办公室 | **PC 必须** |

### 7.2 载体对比（制造场景）

| 载体 | 优点 | 缺点 | 评分 |
|------|------|------|------|
| 钉钉/企微微应用 | 免安装、SSO、消息推送、工厂已普及 | 语音/UI 受限 | 工人/主管 **9/10** |
| 微信小程序 | 相机好、灵活 | 需单独推广 | 无钉钉时 **8/10** |
| 原生 App | 离线、硬件能力强 | 工人不愿装、运维贵 | 特定厂 **7/10** |
| PC Web | 复杂表单、配置、对账 | 车间不适用 | 财务/IT **10/10** |
| 车间平板/工位屏 | 固定点位、大 UI | 需硬件 | 可选 **8/10** |

### 7.3 推荐方案：「1 + 2 + 1」

```text
主入口（约 80% 日常）  →  钉钉 或 企微 微应用（H5）
管理入口（配置/财务）  →  PC Web 管理台
补充入口（可选）        →  车间工位屏 / 微信小程序
```

**Phase 1**：不做独立 App；实施前先确认客户用钉钉还是企微，**只认证一个 IM**。

### 7.4 为什么主入口选钉钉/企微

1. 工厂已在用，工人无需新装 App  
2. 组织架构、角色可直接同步  
3. 待办消息即入口（报工待确认）  
4. H5 一套代码，双端适配  
5. 客户 IT 与采购接受度高  

### 7.5 为什么必须 PC Web

- Graph freeze、Rule 配置、对账、审计导出  
- 财务批量确认、实施工作台  
- 老板多维度分析

## 九、技术架构：多端统一后端

```text
统一 Backend（Graph / Rule / Execution / License / Audit）
  ↓
统一 BFF + Agent API + License 校验
  ↓
多端 Client（同一套 API，不同 UI 壳）
  ├── apps/h5-worker        → 钉钉/企微 WebView
  ├── apps/web-admin        → PC 管理台
  └── optional: mini-program / kiosk
```

**原则**：业务逻辑与模块授权在服务端；端只是壳。

## 十、License 服务（Phase 2 纳入）

### 租户 License 数据结构（规划）

```json
{
  "tenant_id": "factory-hasen-001",
  "plan": "pro",
  "graph_packs": ["graph-work-report", "graph-production-query"],
  "capability_packs": ["cap-report-l2", "cap-read-l0"],
  "connector_packs": ["conn-erp-kingdee-write", "conn-erp-kingdee-read"],
  "im_connector": "conn-dingtalk",
  "factory_count": 1,
  "expires_at": "2027-06-16"
}
```

### 校验点

- Agent Orchestrator 加载 Module 前校验 `capability_packs`  
- Execution Service 执行 DSL 前校验 `capability_packs` + `connector_packs`  
- 未授权返回 `403 MODULE_NOT_LICENSED`

## 十一、销售与实施口径

| 对外说法 | 实际交付 |
|----------|----------|
| 「钉钉里的工厂助手」 | H5 + 已购 Graph/Capability |
| 「免费查询」 | L0 + 限量 |
| 「报工上线（Path A）」 | graph-work-report + cap-report-l2 + ERP write Connector + 实施 |
| 「报工上线（Path B）」 | graph-work-report + cap-report-l2 + MES Connector + 实施 |
| 「接金蝶」 | conn-erp-kingdee-read/write 分开报价 |

**销售单位**：Graph Pack + Capability Pack + Connector + 厂数，不是「一个 App 年费」。

## 十三、一句话

> **FactoryOS 按 Graph / Capability / Connector 三层模块化授权；免费占 L0 只读入口，收费占写操作与连接器；工人/主管走钉钉或企微 H5，财务/IT/实施走 PC Web，Phase 1 不做独立 App。**
