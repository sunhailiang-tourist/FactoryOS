## 〇、一年冲刺总览（主口径）

```text
M1      双轨研发 6 周 → Gate 0（内核 + 0 智接入并行）
M2–M3   灯塔 A D1 结案（客户 ≤90 天）
M4      灯塔 B 复制 2～3 周
M5–M8   第 3 家 + D2 Pack + License DB
M9–M12  3～5 家 tenant · Pack 目录化 · 订阅为主 → 初具规模
```

人力：**2～3 人研发并行**（A 轨内核 / B 轨 Connector+体验）；业务侧 Graph workshop 与 **Path A ERP 样例**从 W1 并行。

下文 **Y1/Y2/Y3** 保留为 **阶段能力标签**（卖什么、技术成熟度），**日历时间以 `14` 为准**。

## 一、路线图总览

```text
                    商业主线                          技术主线
Y1（2026–2027）   灯塔 D1 + Q0 试用转化              Modular Monolith + 通病内核
Y2（2027–2028）   D2 扩展 + 第 2–3 家复制           License + Override + Pack Registry
Y3（2028–2029）   ARR 规模化 + 渠道/伙伴             多 tenant + 计量 + MCP 生态深化
```

**原则**：每一年都有 **可卖 SKU** 与 **可部署制品** 同时交付；技术 Phase 不为「未来愿景」空转，商业里程碑不为「没有产品」讲故事。

## 二、收入四层模型（L0–L3）

| 层 | 名称 | 卖什么 | 典型 SKU | 最早可售 |
|----|------|--------|----------|----------|
| **L0** | 平台订阅 | 运行底座、审计、对账、租户 | Platform Starter / Pro / Enterprise | Y1 Q2（灯塔后） |
| **L1** | D1 通病包 | Graph + Capability + 首期 Connector | Path A: `conn-erp-*-write`；Path B: `conn-mes-*` | Y1 Q1（试用） |
| **L2** | 扩展 Pack | D2 沉淀的 Skill / Graph / Connector | `skill-piece-rate-draft` 等 | Y2 |
| **L3** | Override / 定制 | 厂/线/设备级配置化差异 | Scope Override + 实施人天 | Y2 起 |

**禁止**：把 L3 定制写进 Platform-L0 内核；≥70% D2 须降级为 L2 Pack（见 08 R3/R4）。

## 五、三年里程碑矩阵

### Y1（2026–2027）：地基可卖、灯塔可签

| 季度 | 商业 | 技术（工程 Phase） | 收入验证 |
|------|------|-------------------|----------|
| Q3–Q4 | Phase 0 规格审定；Gate 0 | Schema / ADR / SPEC / AC-BASE-001 | — |
| Q1 | 灯塔 A · Q0 试用启动 | Phase 1：内核 + **Path A ERP 写** + 钉钉 | 试用费 |
| Q2 | 灯塔 A · D1 结案 | Phase 1 收尾；Implementation Package v1 | L0+L1 首单 |
| Q3 | 灯塔 B · D1 复制（Silver import **2–3 周**；全量 **4–8 周**） | Phase 2 启动：读分析 Skill | 复制费 |
| Q4 | 灯塔 A · D2 选 1–2 项 | Phase 2：License 接口 + Override 引擎 | L2 首 Pack |

**Y1 成功标准**：

- ≥1 家 D1 书面结案（≤90 天，研发期除外）
- ≥1 家第二家 D1 ≤8 周（import Package）
- ARR 结构：**订阅 + 实施** 双轨，非纯项目制

### Y2（2027–2028）：Pack 飞轮、多厂复制

| 季度 | 商业 | 技术 | 收入验证 |
|------|------|------|----------|
| H1 | 第 3 家外部/关联厂 D1 | License Service 上线；Pack Registry | Pack 附加费 |
| H2 | D2 Pack 目录对外 | WMS/QMS/OA 只读 Connector 首批 | L2 组合套餐 |
| 全年 | Pro 档位推广 | Integration Studio MVP | NRR > 100% 目标 |

**Y2 成功标准**：

- 共享 Pack ≥10 个（M3+  maturity，见 模块包规格）
- D2 定制 **≥70%** 入库率（08 R4）
- 第 3 家 D1 周期 ≤6 周

### Y3（2028–2029）：规模化与生态位

| 主题 | 商业 | 技术 |
|------|------|------|
| 渠道 | 实施伙伴认证；Connector 认证清单公布 | Partner API + Pack 导入 |
| 计量 | 按厂 / 按写操作 / 按 Connector 可选 | Usage metering + 账单 |
| 开放 | 垂直 Solution 命名（飞轮产出） | MCP Registry 生态化（**Y2 已对外 GA**） |
| 架构 | Enterprise 私有化标品 | 选择性拆服务（license / connector-host） |

**Y3 成功标准**：

- ≥5 家付费 tenant（含哈森体系内）
- Pack 复用率：新 D1 **≥60%** 来自目录选用
- 无「Whale Tax」：单客户定制代码 < 总代码量 15%
