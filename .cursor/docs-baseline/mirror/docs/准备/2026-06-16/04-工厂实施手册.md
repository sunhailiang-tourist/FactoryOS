## 〇、两次交付与七步的关系

| 交付 | 范围 | 七步 | 结案标准 |
|------|------|------|----------|
| **第一次 D1** | 通病包五项 + 报工 Skill | Step 0–7 完整执行 | ≤90 天；Shadow ≥2 周；D1 签字；export Package v1 |
| **第二次 D2** | 工厂特异扩展 | Step 2–7（扩展 Graph/Skill） | D2 书面范围；≥70% Pack 化；export Package v2 |

详见 [08-平台战略与两次交付模型](./08-平台战略与两次交付模型.md) §3、§7。

## 一、「快速接入」的现实预期

| 阶段 | 周期 | 说明 |
|------|------|------|
| 第 1 家（哈森探路） | 6–9 个月 | Connector、Graph、规则、验收模板 |
| 第 2 家（同 ERP + 同行业 + Silver Pack） | **2–3 周 import** + Shadow/UAT | Package import + 差异 workshop |
| 第 2 家（全量 D1 含 Shadow/UAT） | **4–8 周** | 含 Graph 差异确认与 UAT |
| 第 N 家（同 ERP 不同配置） | **6–10 周** | Graph/Rule 差异大，需重确认 |
| 新 ERP 厂商 | **+2–4 周**（S2 Blueprint）或 **+4–8 周**（全量） | 新 Connector Pack |

**快速** = 实施动作标准化、模板可 import，不是「3 天连上任意 ERP」。

---

## 二、标准实施七步

```text
Step 0  售前/准入评估
Step 1  环境接入与摸底（只读）
Step 2  业务链路发现与 Graph 冻结
Step 3  规则与 DSL 绑定
Step 4  沙箱/影子运行
Step 5  试点产线 UAT
Step 6  灰度推广 + 培训
Step 7  运维移交 + 版本管理
```

记忆：**评估 → 接入 → 冻结链路 → 绑规则 → 影子 → 试点 → 推广 → 运维**。

## Step 0：准入评估（1–3 天）

### 0.1 系统清单（必填）

| 项 | 收集内容 |
|----|----------|
| ERP | 厂商、版本、云/本地 |
| MES | 有无、厂商、报工入口 |
| WMS/OA | 系统名称、版本 |
| 接口 | Open API / DB 只读 / 中间表 / ESB |
| IM | 钉钉 / 企微 / 无 |
| 部署 | 客户内网 / 私有云 / 混合 |

### 0.2 准入分级

| 等级 | 条件 | 首期场景 | 周期 |
|------|------|----------|------|
| **S（快）** | 已认证 ERP + 有 API + 有 MES 报工 | 报工 Overlay | 4–6 周 |
| **A** | 同 ERP 但仅 DB 只读 | 只读 + 半自动报工 | 8–12 周 |
| **B** | 不同 ERP 或 Excel 并行严重 | 只读分析为主 | 12–16 周 |
| **C（不接）** | 无 ERP、无编码规范、要求全替换 ERP | — | 推咨询或远期产品 B |

**原则**：只销售**已认证 Connector 组合**；C 级不接。

### 0.3 准入评估 Checklist

- [ ] ERP 在 FactoryOS 认证列表内
- [ ] 客户可提供只读凭证或 API 文档
- [ ] 有明确业务负责人可签字 freeze Graph
- [ ] 有 1 条试点产线/车间
- [ ] 明确钉钉或企微（Phase 1 只认证一个 IM）
- [ ] 客户接受「先只读/确认后写」原则
- [ ] 已采购模块 License（Graph Pack + Capability + Connector）

**交付物**：《准入评估报告》+ 等级判定 + 报价范围。

## Step 1：环境接入与摸底（第 1 周）

### 1.1 部署形态

| 模式 | 适用 |
|------|------|
| 私有化单机/集群 | 中大型，数据不出厂（最常见） |
| 私有云 VPC | 集团多工厂，每厂独立 tenant |
| 边缘+中心 | 产线断网场景（后期） |

复制期建议：**每厂独立部署 + 独立 tenant**。

### 1.2 Connector 只读接入

```text
1. 申请 ERP/MES/OA 只读凭证
2. 部署对应 Connector 插件
3. 跑 Discovery Job（表结构、样本单据、接口文档）
4. 输出《系统摸底报告》
5. 本阶段禁止任何写操作
```

### 1.3 OA 首期策略

| 阶段 | OA 范围 |
|------|---------|
| 复制期首期 | 不接写；或只读审批状态 + IM 通知 |
| Phase 2 | 审批状态查询、消息推送 |
| Phase 3 | APPROVAL_SUBMIT 等 L2 写 + 确认 |

**交付物**：《接口摸底报告》+ Connector 连通证明 + 脱敏样本包。

---

## Step 2：业务链路发现与 Graph 冻结（第 2–3 周）

### 2.1 标准流程

```text
1. 选定 1 条试点业务链（如报工）
2. AI + 工具生成 Graph Draft
3. 实施顾问 + 客户业务负责人 workshop（1–3 场）
4. 修正例外（关班、代报工、价格审核等）
5. 客户在 **Integration Studio** 签字 → freeze Graph v1.0
6. checksum 存档
```

### 2.2 加速资产

| 资产 | 作用 |
|------|------|
| 行业 Graph 模板（如鞋服报工 v1.0） | 新客户约 70% 节点复用 |
| ERP 实体映射表 | 金蝶字段 ↔ DSL 参数 |
| 差异清单模板 | 只记录与模板的差异 |

第 2 家同 ERP：workshop 可减至 **1–2 场**。

### 2.3 Graph 冻结 Checklist

- [ ] Draft 经业务负责人逐节点确认
- [ ] 财务/生产主管签字（电子）
- [ ] status = frozen，version 已分配
- [ ] checksum 已生成并存档
- [ ] 与 RuleSet 版本绑定关系已建立

**交付物**：frozen Graph v1.0 导出包 + 签字记录。

---

## Step 3：规则与 DSL 绑定（第 3–4 周）

```text
1. 从 Graph v1.0 导出所需 DSL 子集
2. 加载行业 RuleSet 模板
3. 按客户组织调整（审批人、关班时间、revert 窗口）
4. freeze RuleSet v1.0（绑定 graph_version）
5. 启用 Agent Module（如 work_report_v1）
6. 配置租户 License（已购模块）
```

### Revertibility 矩阵（示例）

| DSL | 可自动 Undo | 条件 |
|-----|-------------|------|
| WORK_REPORT | ✅ | 未关班、未月结、24h 内 |
| PURCHASE_CREATE | ⚠️ | 未审批/未入库 |
| PAYMENT | ❌ | 禁止 Agent L2 |

**交付物**：《规则配置表》+ 《DSL 启用清单》+ 《Revertibility 矩阵》。

---

## Step 4：沙箱 / 影子运行（第 4 周）

```text
1. tenant.shadow_mode=true（或 dry_run 单笔演练）
2. 执行 DSL，status=simulated；Legacy 不变
3. 主管在 Studio/UI 确认影响报告
4. 与 ERP/MES 手工结果对比 ≥10 笔样本（按 Path A/B）
5. 对账 Job 跑通，无 drift
```

**Gate**：影子阶段**零生产写入**；对账 100% 一致才进 UAT。

**交付物**：《影子对账报告》。

---

## Step 5：试点 UAT（第 5–6 周）

### 5.1 范围

- 1 条产线 / 1 个车间 / 20–50 工人
- 1 个 Agent Module
- 1 个 supervisor 确认流

### 5.2 UAT 验收用例（必过 · 按路径参数化）

| # | 验收项 | Path A（ERP 写） | Path B（MES 写） |
|---|--------|------------------|------------------|
| 1 | 工人完成报工，**Legacy 单据正确** | ERP 落账正确 | MES 有单正确 |
| 2 | 主管 24h 内 revert 成功，数量恢复 | ERP read-back 一致 | MES 数量恢复 |
| 3 | 未 freeze Graph 的请求全部被拒 | 同左 | 同左 |
| 4 | Agent 无法绕过 Rule Engine | 同左 | 同左 |
| 5 | 未授权模块的 DSL 被拒绝 | 同左 | 同左 |
| 6 | 全流程 audit 可导出 | 同左 | 同左 |
| 7 | FactoryOS 与 Legacy **逐笔对账无差异** | vs ERP API | vs MES API |
| 8 | 关班后 revert 被 Rule 拒绝 | 同左 | 同左 |
| 9 | 钉钉/企微入口登录与角色正确 | 同左 | 同左 |
| 10 | PC 管理台可查看 Graph 版本与执行记录 | 同左 | 同左 |

> **哈森灯塔 = Path A**；有 MES 厂 = Path B。禁止混用验收口径。

### 5.3 上线切换

```text
影子 → 试点写生产（L2 确认）→ 观察 1–2 周 → 客户签字上线
```

**Gate**：Graph freeze 签字 + UAT 第 2、7 条通过 → 才切生产。

**交付物**：UAT 签字表 + 上线公告。

---

## Step 6：灰度推广（第 7–8 周+）

```text
产线 1 → 产线 2–3 → 全厂
角色：工人 → 主管 → 老板只读
系统（Path A）：ERP 写+读 → 钉钉通知
系统（Path B）：MES 写 → ERP 只读 → OA 通知
```

新场景 = **Graph v1.1 新版本**，不修改 v1.0。

---

## Step 7：运维移交

| 机制 | 内容 |
|------|------|
| 版本管理 | Graph / Rule / Connector 独立版本号 |
| 对账 | 每日 ExecutionRecord ↔ ERP/MES |
| 变更 | 新 Graph draft → 确认 → freeze → 发布窗口 |
| SLA | Connector 断连、ERP 升级补丁 |

**交付物**：《运维手册》+ 对账告警配置 + 联系人清单。

## 五、不同 ERP/OA 组合差异

| 组合 | 路径 | 重点 | 周期 |
|------|------|------|------|
| **金蝶 + 无 MES（哈森）** | **Path A** | ERP 报工 write Pack 优先 | S：4–6 周 |
| 金蝶 + 自研 MES | Path B | MES 报工 Connector 优先 | S：4–6 周 |
| 用友 + 无 MES | Path A | 报工走 ERP 生产模块 | A：8–10 周 |
| ERP + 钉钉 | Path A/B | OA 首期：通知 + 只读 | +1 周 |
| ERP + 泛微/致远 | Path B+ | 审批 API 复杂，写操作后期 | 首期不接写 |
| ERP + Excel 重 | Path C 候选 | Graph 含 Excel 源，Discovery 难 | B 级 |

## 八、一句话

> **实施 = 评估 → 只读摸底 → Graph/Rule 冻结 → 影子对账 → UAT → 推广；快速复制靠认证 Connector + 行业模板 + Integration Studio，不是 Agent 自动猜业务。**
