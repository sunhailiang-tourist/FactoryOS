## 使用说明

**本版目标**：让老板离场能复述 **五句话**——

```text
1. 干什么   钉钉 + ERP 报工 Overlay，不换 ERP；无系统厂同内核另一条线
2. 为什么不数仓  先赚「敢写库+有人用」；数仓是客户信任后的选配
3. 怎么干   拍板 → 内核 6～8 周 → 哈森 ERP+钉钉 → 双线先锋 → 复制飞轮
4. 花多少   软硬件 **~5.4～9.1 万/年**（**已含**阿里云，见 `12`）+ 人力 2 人起
5. 资产从哪来  每部署一家，现场提炼老系统核心业务 → Graph/Pack；第二家起越来越快
```

**叙事原则**：**递进层次**——是什么 → 为什么这样选 → 两条先锋线 → 路径 → 资产与壁垒 → 增长 → 成本 → 拍板  
**不讲**：12 层架构细节、技术栈表（正文）；**备查附录 E～G** 列全资源与架构图，现场按需翻页

# 第一块｜是什么（Layer 1）

## 第 2 页｜我们是什么：Overlay 一句话

**屏上标题**：「不换 ERP 的执行 OS——这就是 Overlay」

| ✅ Overlay 是 | ❌ 不是 |
|---------------|---------|
| 接在 **ERP / 钉钉** 上面的 **执行入口 + 门禁** | 替换金蝶 / 用友 |
| 工人 **钉钉** 说/扫 → 主管确认 → **写 ERP** | 把 ERP 抄进我们数仓 |
| **Graph + Rule + Audit + Revert + 对账** | 聊天机器人 |
| 按 **Pack** 模块化卖、可复制 | 每家从零定制项目 |

**一张图（建议）**

```text
  工人/主管  →  【FactoryOS】  →  确认·审计·撤回
                    ↓ 写
              【客户 ERP】← 账本仍在这里
              【钉钉】   ← 入口与待办
```

**讲稿**：Overlay = **在上面叠一层**，账本不动，体验和执行在我们这里。

---

## 第 3 页｜先打哪类场景：生产通病（D1）

**屏上标题**：「先做生产的通病，不做全厂全链」

**为什么先做生产报工？**

| 原因 | 说明 |
|------|------|
| 痛点 universal | 工人不爱用系统、主管怕写错、账实对不上——**家家都有** |
| 价值可感知 | 30 秒报工 vs 10 分钟填表，**老板一周就能看见** |
| 风险可控 | 只动产量写入，**不动财务/autopay** |
| 可验收 | D1 五项 + **≤90 天**书面结案 |

**D1 只做五项（屏上大字）**

```text
只读查询 → 受控写入 → 全链路审计 → 可 Revert → 每日对账
```

**哈森首期落点**：**ERP 报工 + 钉钉**（无独立 MES）。

**不讲**：财务 AI、全厂 APS、Day1 数仓。

# 第二块｜为什么这样选（Layer 2～3）

## 第 4 页｜为什么先 Overlay，不先把 ERP/OA 收成数仓？

**屏上标题**：「先叠层，不搬库——两条路线的差别」

| | **Overlay（我们现在）** | **数仓化 / 替换（不做主线）** |
|--|-------------------------|--------------------------------|
| **卖什么** | 敢写、可撤、有人用的 **执行** | 数据汇聚、分析平台 |
| **ERP 角色** | **权威账本（L0）** | 逐步变成被动库 |
| **客户阻力** | 低：「不换 ERP」 | 高：IT/财务/金蝶渠道 |
| **交付** | 6～8 周内核 + 4～5 月首家 | 常变 12 月+ 集成项目 |
| **复制** | import **Pack** | 每家一套 ETL |

**三档位（一图，Y1 只指档位 1）**

```text
档位1  Y1～   控制面上浮   ERP 仍是账本 · 我们管入口     ← 现在
档位2  Y2+    读分析加深   只读看板 Pack
档位3  选配   数据池       客户单独立项 · 非默认承诺
```

**讲稿**：**不是永远不做数据加深**，而是 **没有信任和结案，客户不会心甘情愿把 ERP 当数仓用**。

---

## 第 5 页｜什么时候，客户才心甘情愿让 Legacy「下沉」？

**屏上标题**：「数仓下沉的前提——信任先于搬迁」

**必须同时满足（缺一则不做档位 3）**

| # | 条件 |
|---|------|
| 1 | **≥1 家 D1 书面结案**（Shadow、Revert、对账零 unexplained drift） |
| 2 | 工人/主管 **日常只走 FactoryOS**，Legacy 少人登录 |
| 3 | 客户 **书面采购** Layer 3 / 分析模块（单独合同） |
| 4 | 对账证明：**OS 记录 ↔ ERP 长期一致** |

**下沉的真实含义（纠正误解）**

```text
≠ 把 ERP 全量抄到我们云上
= 人不再点 ERP 界面 · 账本仍可留在 ERP · 我们持执行证明与可选缓存
```

**讲稿**：心甘情愿 = **先用怕了、用信了**，不是销售话术逼迁。

---

## 第 6 页｜双线先锋：Overlay + 无系统，为什么并行？

**屏上标题**：「同一内核，两条先锋线——扩大可打市场」

| 先锋线 | 客户 | 接法 | 何时启动 |
|--------|------|------|----------|
| **A · 有系统 Overlay** | 哈森等（**ERP + 钉钉**） | `conn-erp` 写+读 + 钉钉 | **内核 Gate 0 后立即** |
| **B · 无系统 Lite** | 中小厂无 ERP/MES | 同内核 + `conn-mes-builtin` 内置账本 | **内核稳 + 哈森 Shadow 后**（约 M4 起） |

**为什么双线，不是二选一？**

```text
同一套 OS（Graph/Rule/Revert/Audit）——只换 Connector 目标
A 线验证：有 ERP 的「敢写库」
B 线验证：无系统的「低成本拥有生产体系」
双线 Pack 都沉淀 → 第三家起 import，不翻倍研发
```

**为什么不是 Day1 三线齐开？**

- **内核必须先硬**（6～8 周）  
- B 线 **晚 1～2 月**，不抢哈森 ERP 接口资源  
- 人力：**2 人内核 → Gate 0 后 +1 接 A → M4 可选 +1 开 B**

**配图**：中间「OS 内核」+ 左 A（ERP+钉钉）+ 右 B（内置账本）

# 第四块｜资产、壁垒、增长（Layer 5～6）

## 第 9 页｜场景定制 vs 数据飞轮（关系讲清）

**屏上标题**：「定制要变成 Pack，不是变成债务」

```text
          客户现场定制（D2）
                 ↓
          ≥70% 沉淀为 Pack
                 ↓
          export Implementation Package
                 ↓
          下一家 import → 周期 2～3 周
                 ↓
          Pack 目录越来越厚 = 飞轮
```

| | **场景定制（受控）** | **数据飞轮（资产）** |
|--|----------------------|----------------------|
| 是什么 | 某厂特异逻辑（计件、班次） | **可卖的 Pack + 对账/Revert 模板** |
| 放哪 | Override / L2 Pack | Pack Registry + export |
| 铁律 | **≥70% 入库**，否则不算交付完成 | 无 export = 交付未完成 |
| 不做什么 | 改 execution 内核 | **不**默认托管客户全量 ERP 数据 |

**讲稿**：定制是 **inputs**；飞轮是 **outputs**——越交付越轻，不是越交付越胖。

**顶级战略（大白话，可原句讲）**：**每部署一家厂，就是在现场飞速提炼老系统核心业务，变成我们的 Graph 和 Pack——数字资产；第一家考古，第二家 import。**（见 `08` §1.4，≠ 扒全库做数仓）

## 第 10 页｜核心资产与壁垒：如何一步步长出来？

**屏上标题**：「壁垒不是模型，是机制 + 复利资产」

**成长阶梯（递进讲）**

| 阶 | 时间 | 沉淀什么 | 壁垒 |
|----|------|----------|------|
| **1** | Gate 0 | Graph/Rule/Revert/**Audit 机制** | 敢写库 |
| **2** | 首家 D1 | 报工 Graph + ERP Connector + **Package v1** | 有人用、可结案 |
| **3** | D2 + 复制 | 扩展 Pack + **import 4～8 周→2～3 周** | 复制速度 |
| **4** | 12 月 | Pack 目录 + 多 tenant License | 规模毛利 |
| **5** | 选配 | Layer 3 分析 / 数仓加深 | 客户信任后加价 |

**四层壁垒（屏上）**

```text
1. 产品机制  frozen Graph + Rule + 写路径唯一 + Revert + 对账
2. 工程纪律  Agent 永不直写账本（11 条红线）
3. 交付纪律  Shadow / 90 天结案 / D1 五项
4. 复利资产  Pack 目录 + Implementation Package
```

**讲稿**：外挂 Agent **1～2 周**能 demo；这套 OS **6 个月起**，难在 **机制+纪律+Pack**。

## 第 11 页｜增长从哪来：通病可复制 + 无系统优雅接入

**屏上标题**：「两条先锋线汇合后，客户数怎么涨」

```text
                    ┌─ A 有 ERP ──→ 哈森 → 同行业 import（2～3 周）
  内核 + Pack 飞轮 ─┤
                    └─ B 无系统 ──→ B-Lite 试点 → 同 Pack 机制，换 Connector

         第 1 家（重）          第 2～3 家（轻）           第 4～5 家（选 Pack）
              │                      │                         │
         实施+定制              mostly import              目录勾选+实施
              ↓                      ↓                         ↓
         订阅 ARR ↑              NRR > 100%              订阅为主 7:3 目标
```

**老板可预期的增长逻辑**

| 里程碑 | 商业信号 |
|--------|----------|
| 首家 D1 结案 | 证明 **能收钱、能交付** |
| 第二家 ≤3 周 | 证明 **不是项目制** |
| B 线 1 家试点 | 证明 **TAM 扩大**（无系统厂也能进） |
| 12 月 3～5 tenant | **初具规模**（原 Y3 目标前移） |

**定价**：首家 D1 后回填；结构 = **L0 订阅 + L1 Pack + 实施**。

| | **测试域** | **生产域** |
|--|------------|------------|
| **干什么** | 代码仓库 · CI · staging 验收 · API 文档 | **只跑** FactoryOS 业务 |
| **核心组件** | GitLab + Runner + staging + Redoc | SLB + API + RDS + Redis + OSS |
| **数据库** | RDS 测试（可重建） | RDS 生产（15 天备份） |
| **谁用** | 研发 | 工人/主管（钉钉） |
| **禁止** | — | GitLab、YApi、自建 PG |

**测试域四类部署（一句话）**：

```text
① GitLab      代码放哪
② Runner+staging  push 后自动测、预发布
③ Redoc       接口契约对内查阅（Registry published + export，不用 YApi）
④ RDS 测试    联调数据
```

# 附录 F｜三张架构图导览（备查 · 建议 2～3 页）

## F-1｜系统架构图——平台分层与扩展

```text
L3 Harness（钉钉/PC）→ L2 Agent/Skills → L0 信任内核 → L1 Connector → Data-L0 客户账本
```

**老板只需记住**：

| 层 | 一句话 |
|----|--------|
| L3 | 工人/主管用的界面 |
| L2 | AI 只产出计划，**不能直写 ERP** |
| L0 | Graph/Rule/执行/审计——**全行业复用的宪法** |
| L1 | 接 ERP/MES/WMS… **换 Pack 不换内核** |
| L0 数据 | 账本仍在客户 ERP |

**双切入（图内 Phase1 标注为通用路径，哈森另线）**：

| 路径 | Connector |
|------|-----------|
| 哈森 | ERP 读+写 + 钉钉 |
| 有 MES 厂 | MES 写 + ERP 读 |
| B-Lite | 内置账本 |

## F-2｜技术架构图——模块与 AI 边界

```text
仅 agent_orchestrator 用 LLM → 输出 DSL JSON
graph / rule / execution / audit / connector → 零 LLM
唯一写路径：execution → connector_sdk → Legacy
```

**制造业 10 类系统**：MES ERP WMS OA CRM CSM APS PLM QMS + mock——**新厂商加 Pack，不加内核 if 分支**。

## F-3｜数据架构图——数据放哪

```text
L0 客户 ERP/MES（权威，不在我们 PG）
L1 运行数据（Graph/Audit/执行记录）→ RDS 必做
L2 缓存（TTL，最小字段）→ 按需
L3 数仓/分析 → 选配，非默认
媒体（语音/图）→ OSS，不进 PG
```
