# AI 编码工程规则

## 项目本质

FactoryOS 是制造业 AI **执行平台底座**（Overlay on ERP/MES），不是聊天机器人。

**产品 B Lite（无 ERP）**：同一内核 + `conn-mes-builtin` 写内置 PG 账本；规格见 `文档/规格说明/产品B-Lite无ERP内置账本.md`；Y1 默认不实现，拍板后见 `准备/2026-06-16/13-产品B-Lite应急开发指引.md`。

**双极架构**（不可偏废）：

| 极 | 承诺 | 主要落位 |
|----|------|----------|
| **终端智能极** | 0 智、多模态（说/拍/扫）；Harness 确认门 | `server/api` modules（perception/harness 预留） · `src/apps/h5-worker` |
| **内核门禁极** | frozen Graph、Rule、DSL、Audit、Revert | `os_core/*` Platform-L0 |

当前阶段：**只建底座**，不绑具体工厂业务 Graph。

## 架构红线（ADR-002 R-01～R-11，违反即拒绝）

1. Agent / `agent_orchestrator` **禁止** 调用 Connector.write 或直接写数据库（R-01）
2. 所有 Legacy **写** 仅经 `os_core/execution_service`（R-02）
3. Graph `status !== frozen` 时 **禁止** L2 写 DSL 执行（R-03）
4. Rule Engine **不可绕过**；默认 deny（R-04）
5. 每个 L2 DSL **必须** 有 Compensator（R-05）
6. 每次写操作 **必须** ExecutionRecord + Audit（R-06）
7. **禁止** 原地修改 frozen Graph / RuleSet（R-08）
8. **禁止** Agent 自动 freeze Graph（R-09）
9. 未授权 Pack / DSL / Connector → 403 + Audit（R-10）
10. 多模态/高置信识别 **不得** 跳过 Harness 确认门与 Rule Engine（R-11）

## 优先级

```text
可靠性 > 安全性 > 可追溯 > 可回滚 > AI 能力
```

## 技术栈（ADR-001）

| 层 | 选型 |
|----|------|
| 后端 | **Python 3.12+** · FastAPI · Uvicorn · SQLAlchemy 2 async · PostgreSQL 16 |
| AI | **LangGraph** + **LiteLLM**（仅 `agent_orchestrator`） |
| 契约 | **Pydantic v2** 对齐 `contracts/schemas/` JSON Schema |
| 测试 | **pytest** + pytest-asyncio + httpx ASGI |
| 类型 | **pyright** 或 **mypy** strict（os_core） |
| 前端（Phase 1 后半） | React 18 + **TypeScript** + Vite（`web-admin` / `h5-worker`） |
| 包管理 | **uv** workspace |

## 模块边界

Python 包名 **snake_case**，见 [命名约定](../文档/架构/命名约定.md) 与 ADR-001 §3：

```text
os_core/shared_contracts/              — Pydantic 模型、Schema、错误码
os_core/graph_service/       — 版本、freeze、校验（无 Connector 写）
os_core/rule_engine/         — 授权（无 Connector 写）
os_core/execution_service/   — 唯一写 Legacy 路径
os_core/audit_service/       — append-only
os_core/agent_orchestrator/  — LangGraph FSM → DSL 计划（无写）
os_core/connector_sdk/       — httpx 适配 read/write
os_core/license_service/     — Pack 授权（Phase 1 stub）
os_core/mcp_gateway/         — MCP JSON-RPC（Y1 末内部 GA；W7 stub）
src/server/api/                 — FastAPI 入口（含 perception / harness 路由）
```

## 编码绝对门禁（强制）

- **用户未确认前禁止** 编写 `os_core/`、`apps/` 业务 `.py` / `.tsx` 实现
- 每个项目根 **必须先有 README.md**（是什么 / 功能 / 不负责 / 上下游）
- **中文注释**：文件头（作用·业务·上下游）+ 函数 docstring + 字段级业务含义
- **禁止代码重复**：`scripts/check_code_redundancy.py`；`server/api` 不得重复 `os_core` 能力
- 确认口令：**「确认编码门禁，开始 W1」**

## 编码规范

- **后端 Python**：pyright/mypy strict；禁止 `Any` 绕过类型
- **前端 TypeScript**：strict mode（仅 `src/apps/web-admin`、`src/apps/h5-worker`）
- 先写 contract/integration test（对照 AC-BASE-001 / **AC-UX-001** 用例 ID）
- API 请求/响应对照 `contracts/schemas/` 与 **`contracts/openapi/工厂操作系统-v1.1.yaml`**（info.version **1.1.1**）
- 新 DSL 动词：更新 DSL 规格 + registry + 测试
- Connector 参数用明确 Pydantic 类型

## Pack / License / Override（ADR-003）

- 新能力 **先登记** 能力追溯矩阵 + 模块包规格，再 Coding
- 未授权 Pack → 403 `MODULE_NOT_LICENSED`
- 客户差异走覆盖配置规格；**禁止** `if tenant_id == "xxx"` 内核分支
- 全表含 `tenant_id`；ExecutionRecord 可选 `scope_id`
- D1 Starter 含 `cap-perception-v1`（多模态入口）

## 禁止事项

- 不要实现具体 MES 厂商逻辑，除非 `文档/连接器/` 有样例
- Legacy system 枚举 10 类 + mock；Phase 1 仅 mes+erp，其余 Registry stub
- 不要并行开发 Boss Agent、全 ERP 写集成、独立 App
- 不要跳过 Graph freeze 门禁「方便测试」
- 不要跳过高置信多模态的 Harness 确认门（R-11）
- 不要修改 `准备/` 下文档，除非用户明确要求
