## 1. 产品承诺（对外可述）

> **终端：说话、拍照、扫码就能办事——0 智、极低培训。**  
> **内核：未 freeze 不写、无 Rule 不写、无确认不写、写了必可审计与 Revert。**  
> **结果：老系统仍在，人不再频繁点进去；数据链路随执行同步，对账可证。**

二者 **同时成立** 才是 FactoryOS，不是二选一。

---

## 2. 双极对照表

| 维度 | 终端智能极 | 内核门禁极 |
|------|------------|------------|
| 用户感知 | 简单、快、像助手 | 放心、可追责、敢用 |
| 技术载体 | H5/Web + 感知 + LangGraph Skill | Graph / Rule / DSL / Execution / Audit |
| 失败时 | fallback 表单/扫码 | 403/409 + Audit |
| 优化目标 | 少点击、少记忆、少培训 | 零越权、零 silent write、零 unexplained drift |
| 验收 | UX-001 | BASE-001 + MVP-001 |

## 3. Harness Loop（体验 × 门禁合一）

```text
感知 Perceive     语音 / 图 / 视频 / 扫码 / 文字
    ↓
理解 Understand   Skill FSM + 填槽 → DSL 计划（仅计划）
    ↓
授权 Authorize    License Pack + Rule Engine（默认 deny）
    ↓
沙箱 Simulate     dry_run 影响报告（L2 写前）
    ↓
确认 Confirm      人点确认 / 主管待办（L2 必须）
    ↓
执行 Execute      execution_service → Connector
    ↓
验证 Verify       read-back + 对账 Job
    ↓
记录 Record       ExecutionRecord + Audit + perception_refs
```

**智能越强，Confirm 卡片越要清晰**（展示：改什么、改多少、能否 Revert）。

## 4. 三端体验定义

### 4.1 工人端（`apps/h5-worker` · 钉钉/企微）

| 原则 | 说明 |
|------|------|
| **0 智** | 默认只显示「当前工单」相关动作，无 ERP 菜单树 |
| **三键内完成** | 扫/说/拍 → 确认数量 → 提交（待主管） |
| **噪声友好** | 语音失败 → 扫码；识别低置信 → 大数字键盘 |
| **即时反馈** | 「已提交，待班长确认」+ 可查本单进度（D2） |

**替代的老系统负担**：MES 多屏报工、纸质工票、微信报数。

### 4.2 管理端（H5 待办 + `web-admin`）

| 原则 | 说明 |
|------|------|
| **实时确认** | IM 待办推送；列表批量通过/驳回（D2） |
| **一眼看清** | 待确认 / 今日异常 / 对账 drift 三栏 |
| **说查就查** | 「线 3 今天报工多少」→ L0 只读，不写 |
| **Revert 可达** | 24h 内一键撤销 + 原因登记 |

**替代的老系统负担**：ERP+MES 切换、Excel 汇总、电话追报工。

### 4.3 老板端（H5 摘要 + PC 看板）

| 原则 | 说明 |
|------|------|
| **链路摘要** | 订单/产线/延期风险 **只读** 自然语言问数（Phase 2 全链） |
| **不误导** | 摘要附数据来源与 freshness；无数据则明说 |
| **零误写** | 老板端 **默认无 L2**；建议仅 L0/L1 |

**替代的老系统负担**：等报表、开会问进度、多系统对口径。

## 5. 效率 KPI（哈森实测回填）

| 指标 | 测量方式 | D1 目标（参考） | 说明 |
|------|----------|-----------------|------|
| 工人单次报工耗时 | 现场抽样 | **≤ 30s**（P50） | UX-001 |
| 主管日确认耗时 | 待办处理时长 | 较基线 **≥ 5×** 提升 | 需 before 数据 |
| 对账人工 | 人天/月 | 较基线 **≥ 10×** 减少 | 对账 Job |
| 培训上岗 | 新工人独立报工 | **≤ 10 分钟** | 0 智验收 |
| 数据同步延迟 | 报工→MES 可见 | **≤ 60s**（P95） | 非全厂实时数仓 |

「数十倍」仅对 **人肉搬运/对账/多屏点击** 类工作承诺；全厂秒级一致不承诺。

## 8. 反模式（禁止）

| 反模式 | 后果 |
|--------|------|
| 高置信 VLM 自动写 MES | 违反 L2 确认；事故 |
| 只做 PC 不做 H5 | 工人端 0 智失败 |
| 老板端开放写库存/财务 | 违反 ADR-002 |
| 感知成功即执行 | 绕过 Harness Confirm |
| 无 fallback 仅语音 | 车间不可用 |
