# 验收用例 UX-001：终端体验与多模态

| 版本 | v0.2.0 |
|------|--------|
| 范围 | Platform-L3 Harness + 感知层；**工厂 Graph frozen 后**与 AC-MVP-001 一并执行 |
| 路径 | UAT 按 **Path A（ERP 落账）** 或 **Path B（MES 有单）** 参数化（ADR-005） |
| 通过标准 | D1 体验 Gate：下列 **P0** 全部通过 + 内核 BASE-001 / MVP 无回归 |

**说明**：本用例验证 **「终端智能极」**；内核 **「门禁极」** 仍以 [BASE-001](./验收用例-BASE-001-平台底座.md) 为准。二者 **缺一不可**。

---

## 一、感知与 Fallback（P0）

| ID | 用例 | 步骤 | 期望 |
|----|------|------|------|
| M-01 | 语音报工主路径 | 工人语音「工单 X 完成 50」→ 确认卡 | DSL 计划正确；未确认前 **Legacy 无变化** |
| M-02 | 低置信澄清 | 模拟 confidence &lt; 0.85 | 进入澄清或 fallback，**不执行** |
| M-03 | 扫码 fallback | 关闭/失败 ASR 后扫码工单 | 同等完成报工计划 |
| M-04 | 大按钮 fallback | tap 选最近工单 + 数量 | 可完成，无需语音 |
| M-05 | 拍照建议不直写 | 拍照识别数量 → 展示建议值 | 用户改数/确认后才进 Harness |
| M-06 | 感知日志 | 执行后查 Audit | 含 `perception_refs`，无密钥 |

## 二、Harness 确认门（P0）

| ID | 用例 | 步骤 | 期望 |
|----|------|------|------|
| H-01 | 未确认不执行 | 仅 `/v1/agent/plan` | Legacy 无写 |
| H-02 | 确认后执行 | plan → confirm → execute | **Legacy 落账/有单** + ExecutionRecord |
| H-03 | dry_run / shadow | dry_run=true 或 tenant.shadow_mode | Legacy 不变；**status=simulated** |
| H-04 | 主管驳回 | 主管点驳回 | 无写；Audit 有记录 |
| H-05 | 越权确认 | 工人角色点他人待办 | 403 RULE_DENIED |

## 三、三端场景（P0 / P1）

| ID | 用例 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| T-01 | 工人 30 秒报工 | 现场计时 P50 | ≤ 30s | P0 |
| T-02 | 新工人 10 分钟上岗 | 无 Legacy 系统培训，仅 H5 引导 | 独立完成 1 笔报工 | P0 |
| T-03 | 主管 IM 待办 | 钉钉收到待办 → 点确认 | **Legacy 更新**（Path A: ERP / Path B: MES） | P0 |
| T-04 | 主管语音只读查数 | 「线 3 今日产量」 | L0 返回；无写 | P1 |
| T-05 | 老板摘要卡片 | 推送日报只读 | 无 L2 按钮 | P1 |
| T-06 | 对账零 unexplained drift | 日报工 24h 后对账 Job | ok 或告警有明细 | P0 |
