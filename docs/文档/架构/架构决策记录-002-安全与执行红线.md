# 架构决策记录 002：安全与执行红线

| 字段 | 值 |
|------|-----|
| 状态 | **Accepted（底座 Phase 0）** |
| 日期 | 2026-06-16 |
| 来源 | `.cursor/rules/工厂操作系统.md` 须与本 ADR 一致 |

## 优先级（不可颠倒）

```text
可靠性 > 安全性 > 可追溯 > 可回滚 > AI 能力
```

## 红线（违反即拒绝合并）

| ID | 红线 |
|----|------|
| R-01 | Agent / Orchestrator **不得** 直接调用 Connector 写接口或写数据库 |
| R-02 | 所有写 Legacy 的路径 **仅** 经 `execution_service` |
| R-03 | `graph.status !== frozen` 时，**禁止** 任何 L2+ 写 DSL 执行 |
| R-04 | Rule Engine **不可绕过**；无匹配 allow 规则则 deny |
| R-05 | 每个写 DSL **必须** 注册 Compensator；无 Compensator 不得注册为 L2 写动作 |
| R-06 | 每次写操作 **必须** 产生 ExecutionRecord + Audit 条目 |
| R-07 | 生产写操作 **必须** 经 dry_run 或 Shadow 模式验证路径（租户配置） |
| R-08 | Graph / Rule **禁止** 原地修改 frozen 版本；仅允许新版本 |
| R-09 | AI **不得** 自动将 Graph 设为 frozen；freeze 仅人工/API 显式操作 |
| R-10 | 未授权 Module / DSL / Connector **硬拒绝**（HTTP 403 + 审计） |
| R-11 | 多模态/高置信识别 **不得** 跳过 Harness 确认门与 Rule Engine（见 [感知与多模态入口](../规格说明/感知与多模态入口.md)） |

## Agent 安全级别

| 级别 | 模式 | 底座 Phase 1 支持 |
|------|------|-------------------|
| L0 | 只读 | ✅ |
| L1 | 建议，不执行 | ✅（接口预留） |
| L2 | 执行 + 确认 | ✅ |
| L3 | 低风险自动 | ❌ Phase 2+ |

## 首期禁止的 DSL 类别（即使注册也不默认启用）

- 财务：付款、成本结转、自动记账
- 库存：STOCK_ADJUST 无确认
- 审批：APPROVAL_SUBMIT 无 L2 确认

## CI / Code Review 检查点

1. `grep` 禁止 `connector.*write` 出现在 `agent_orchestrator` / `graph_service` / `rule_engine`
2. execution 单测：unfrozen graph → 期望 409
3. execution 单测：无 rule match → 期望 403
4. 每个写 DSL 单测：revert 后状态一致（mock connector）
5. harness 单测：仅有 `dsl_plan` 无 `confirm` → Legacy 无写（R-11）
