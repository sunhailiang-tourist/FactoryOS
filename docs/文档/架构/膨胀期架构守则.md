# 膨胀期架构守则

> 版本：**v1.0.0** | 日期：2026-06-16  
> 状态：**Accepted**（Gate 0 / Core 1.0 Coding 前必读）  
> 关联：ADR-001 · ADR-004 · [os_core-public-api](./os_core-public-api.md) · [过渡架构-四态演进](./过渡架构-四态演进.md)

---

## 1. 目的

在 **Modular Monolith** 形态下，保证 tag `core-v1.0.0` 后 **L0 内核短期不动**，而 GIP / 多工厂 / MCP 膨胀时 **integration/ 与前端可独立演进**，且模块边界可被 CI 强制。

对标：2026 Modular Monolith 共识 — **边界先于分布**（ArchUnit / import-linter / fitness function）。

---

## 2. 模块依赖矩阵（os_core/）

| 模块 | 可调 | 禁止直接 import |
|------|------|-----------------|
| `shared_contracts` | — | 任何 os_core 模块 |
| `audit_service` | `shared_contracts` | 其他业务模块 |
| `graph_service` | `shared_contracts`, `audit_service` | execution, connector 私有 |
| `rule_engine` | `shared_contracts`, `audit_service` | execution, graph 私有 |
| `execution_service` | `shared_contracts`, `audit_service`, `connector_sdk` **公开面**, `rule_engine` **公开面**, `graph_service` **公开面** | agent 私有 |
| `connector_sdk` | `shared_contracts` | execution 私有 |
| `agent_orchestrator` | `shared_contracts` | execution 内部写路径 |
| `license_service` | `shared_contracts` | — |
| `mcp_gateway` | `shared_contracts`, `agent_orchestrator` **公开面** | execution 私有 |

**跨模块调用规则**：

1. 模块间仅通过 **`os_core/<mod>/api.py`（或 `public/`）** 暴露符号；`internal/`、`_*` 禁止跨模块 import。  
2. `apps/api` 仅作 HTTP 适配层，**不得**含业务规则。  
3. `integration/` 遵守 [os_core-public-api](./os_core-public-api.md) §2。

---

## 3. 服务提取触发器（Y3 前默认不拆）

满足 **任一** 条件方可启动 ADR 评审 extract 独立 deployable：

| 触发器 | 阈值（示例） | 候选模块 |
|--------|--------------|----------|
| **独立 scale** | 单模块 CPU > 60% 且其他 < 20%，持续 14d | `mcp_gateway` |
| **独立团队** | ≥2 全职仅维护 integration/GIP | `integration/` 工具链（非 Core） |
| **合规隔离** | 客户合同要求 MCP/Agent 进程级隔离 | `mcp_gateway` |
| **发布节奏** | Core 与 GIP 发布周期冲突且无法 feature flag | 仅 `mcp_gateway` 或 Edge（非 L0） |

**禁止 extract**：`graph_service`、`rule_engine`、`execution_service`、`audit_service` 四者 **不得拆为独立写路径服务**（除非全新 ADR 重写 L0）。

**百级千级**：租户/Cell/Connector 实例/Quota/Outbox 演进见 [ADR-007](./架构决策记录-007-百级千级演进策略.md)；`connector_worker` 池 **仅** 在 ADR-007 §12 触发条件下 extract，且 **不得** 持有写 Legacy 权限。

---

## 4. 内部通信（Core 1.0）

| 场景 | Core 1.0 | Y2+ 可选 |
|------|----------|----------|
| Graph freeze 通知 | 同进程函数调用 / 同步 | Outbox → 消息队列 |
| Execute 完成 | audit append 同步 | 同上 |
| 跨模块查询 | 公开 API 调用 | 不变 |

**原则**：W1 起 event handler 接口预留 [`DomainEvent`](../数据结构/DomainEvent.schema.json) 类型名，**不**引入 Kafka。

---

## 5. CI 健身函数（W1 启用）

| 脚本 | 门禁 |
|------|------|
| `scripts/check_cmv_sync.py` | CMV YAML 结构 + L2 compensator |
| `scripts/check_import_boundaries.py` | §2 矩阵 + integration 公开面 |
| `scripts/check_openapi_schema_refs.py` | OpenAPI `$ref` 可解析 |
| `pytest -m contract` | JSON Schema 样例校验 |
| `pytest tests/integration` | AC-BASE-001 ID 映射 |

`check_import_boundaries.py` 最小行为（W1 实现）：

```text
1. 扫描 os_core/*/*.py import 图
2. 违反 §2 矩阵 → exit 1
3. 扫描 integration/**/*.py 仅允许 public-api 符号
```

---

## 6. Core 变更门禁

| 变更类 | 要求 |
|--------|------|
| `os_core/*` 任意 | 全量 AC-BASE-001 P0 + ADR 修订（G2+） |
| OpenAPI breaking | 新版本路径或 ADR；旧版保留 ≥1  minor |
| 新 CMV 动词 | CMV PR + Schema + OpenAPI MCP tools |
| `integration/*` | Pack contract tests + API smoke |
