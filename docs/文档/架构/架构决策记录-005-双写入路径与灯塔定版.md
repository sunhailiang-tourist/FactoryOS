# 架构决策记录 005：双写入路径与灯塔定版

| 字段 | 值 |
|------|-----|
| 状态 | **Accepted** |
| 日期 | 2026-06-16 |
| 范围 | Phase 1 生产写路径；废止 ADR-000 C-03 单一表述 |
| 依赖 | ADR-001, ADR-004, [14-一年冲刺路线图](../../准备/2026-06-16/14-一年冲刺路线图与并行研发.md) |

## 背景

全库审计发现 **C-03「MES 写 + ERP 只读」** 与 **哈森灯塔「ERP 写+读 + 钉钉、无 MES」** 并存，导致 Gate 0'、UAT、Connector Pack、验收文档口径分裂。  
FactoryOS 内核 **不变**；差异仅在 **L1 Connector Pack 组合** 与 **Data-L0 账本落点**。

## 决策

### 1. 三条接入路径（并存，按客户选型）

| 路径 ID | 名称 | 典型客户 | Data-L0 | 生产写落点 | 读路径 |
|---------|------|----------|---------|------------|--------|
| **Path A** | Overlay · ERP 主导 | **哈森灯塔**（无独立 MES） | 客户 ERP | `conn-erp-{vendor}-write` | ERP + 钉钉 |
| **Path B** | Overlay · MES 主导 | 有 MES 的离散制造厂 | 客户 MES（执行）+ ERP（计划） | `conn-mes-{vendor}` L2 写 | ERP L0 读 |
| **Path C** | B-Lite 内置账本 | 无 ERP/MES（应急） | PG `builtin_ledger` | `conn-mes-builtin` | 内置表 |

**L0 机制三条路径完全相同**：Graph → Rule → Execution → Audit → Revert → 对账。

### 2. 灯塔定版（2026 主路径 = Path A）

| 项 | 定版 |
|----|------|
| 灯塔客户 | 哈森体系 |
| 外部系统 | **ERP 写+读 + Revert**；**钉钉** IM/OA 入口 |
| MES | **不接**；Registry 保留 `mes` enum + stub |
| Gate 0' 样例 | **ERP API 样例** + 报工 Graph frozen（非 MES 样例） |
| AC-MVP-001 对账 | OS 记录 vs **ERP read-back** |

### 3. 平台默认路径（非灯塔客户）

- **Path B** 仍为平台 **通用制造业默认**（有 MES 厂：MES 写 + ERP 读）。
- 销售/实施文档须 **显式标注路径 ID**，禁止混用「Phase 1 = MES 写」而不说明灯塔例外。

### 4. 废止与替换

| 原表述 | 替换为 |
|--------|--------|
| ADR-000 **C-03** 单一「MES 写 + ERP 读」 | **C-03a** Path B 默认；**C-03b** Path A 灯塔；**C-03c** Path C B-Lite |
| Gate 0'「MES API 样例」 | Path A：**ERP** 样例；Path B：MES 样例 |
| UAT「MES 有单」 | 按路径：ERP 落账 / MES 有单 |
| D1 Starter 仅 `conn-mes-*` 写 | 分 **Starter-A**（ERP 写）、**Starter**（MES 写）、**Starter-B-Lite**（内置账本） |

### 5. Pack SKU 对应

| Pack 组合 | 路径 | 商业 SKU |
|-----------|------|----------|
| `conn-erp-*-write` + `conn-dingtalk` + graph-work-report | A | Starter-A（哈森） |
| `conn-mes-*` + `conn-erp-*-read` + graph-work-report | B | Starter（通用） |
| `conn-mes-builtin` + graph-work-report | C | Starter-B-Lite |

### 6. Coding 约束（不变）

- `execution_service` **唯一**写 Legacy；路径差异 **仅** Registry 解析的 Connector Pack。
- **禁止** `if (customer == 'hasen')`；用 tenant License + Pack 绑定。

## 后果

- 正面：全库单一真源；灯塔与平台通用路径可并存叙述。
- 负面：需批量勘误 `03/04/05/000/摘要` 等；CAPABILITY-PACK-MAP 须增 Starter-A。
