## 1. 如何使用

- **销售 / 商务**：按「商业能力」列找 Pack ID 与档位（见 [PACK-CATALOG](../商业/模块包商业目录.md)）
- **研发**：按「os_core/ 模块」列划边界；新能力 **先登记本表再 Coding**
- **实施**：D1 按路径选 §2 通病包；D2 扩展登记 §3 并更新 Pack 成熟度

**规则**：表中每一行 Pack ID 须在 `模块包.md` 有定义；未登记不得对外报价。

**写入路径**（ADR-005）：

| 路径 | 商业 SKU | 生产写 Pack |
|------|----------|-------------|
| Path A | Starter-A | `conn-erp-{vendor}-write` |
| Path B | Starter | `conn-mes-{vendor}` |
| Path C | Starter-B-Lite | `conn-mes-builtin` |

## 5. os_core/ 目录 ↔ 职责（Coding 边界）

> Python 包名 snake_case，见架构决策记录 001 §3。

| Package | 拥有能力 | 禁止 |
|---------|----------|------|
| `graph_service` | Graph CRUD、freeze、checksum、Override 合并 | 写 Legacy |
| `rule_engine` | 授权、条件、Agent 级别 | 写 Legacy |
| `execution_service` | DSL 执行、Saga、Revert、对账 | 业务 UI |
| `audit_service` | append-only 日志 | 改历史 |
| `agent_orchestrator` | LangGraph Skill FSM → DSL 计划 | 写 Legacy、绕过 Rule |
| `connector_sdk` | Legacy 读写适配（httpx） | 业务规则 |
| `license_service` | Pack License、Override 生效域 | 执行 DSL |
| `mcp_gateway` | MCP tools/list、tools/call → DslPlan | 直写 Legacy、绕过 Rule |
| `shared_contracts` | Pydantic 模型、Schema、错误码 | 业务逻辑 |

## 7. 变更流程

1. 新 Pack：在本表加行 → 更新 `模块包.md` → `模块包商业目录.md` → PR  
2. 新 DSL：更新 `DSL执行动词.md` → 绑定 Capability Pack → Rule 模板  
3. 新 Legacy system：**禁止**（10 类 enum 已定）；新 vendor 用 `conn-{system}-{vendor}`  
4. 定制需求：先评估是否 `override-scope`；否则走 D2 Pack 流程（GOVERNANCE）
