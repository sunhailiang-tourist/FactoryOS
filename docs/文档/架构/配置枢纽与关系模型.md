## 0. ADR-008 裁定（2026-06-26）

| 平面 | 真源 | 废止 |
|------|------|------|
| Contract（OpenAPI/Schema/CMV） | `contract_sets` + `contract_artifacts` + publish | `contracts/` 日常编辑 |

**实施界面只认 API + DB**；Git 目录仅为 export/fixture/CI 镜像。

## 1. 设计目标

| 目标 | 手段 |
|------|------|
| **见名知意** | 关系配置文件名、字段名直接表达「谁连谁、用什么 Pack」 |
| **高内聚低耦合** | `os_core/` 冻结；**客户差异进 Registry**（export 至 `integration/`） |
| **低成本迁出** | 迁出 = **Git 目录 + Implementation Package export**；非拆微服务 |
| **单一真源** | 同一关系不在 Graph、Blueprint、DB 三处重复定义 |
| **Registry + Studio** | PostgreSQL Registry 为关系 **published 真源**；Git export 为镜像 |

**废止口径**：「各子项目 0 成本独立迁出」→ **「配置枢纽 + Package 可复制」**。

## 2. 三层配置模型

```text
┌─────────────────────────────────────────────────────────────────┐
│  Layer A · ConnectorDefinition（全局能力，可复用）                  │
│  integration/catalog/{system}/{vendor}/                          │
│  blueprint.yaml · mapping.yaml · contract tests                  │
├─────────────────────────────────────────────────────────────────┤
│  Layer B · ConnectorInstance（租户绑定，项目级）                    │
│  integration/tenants/{tenant_id}/system_relations/*.yaml         │
│  → activate 后落库 connector_instances                           │
├─────────────────────────────────────────────────────────────────┤
│  Layer C · CrossRelation（可选，租户间/系统间协作声明）            │
│  integration/tenants/{tenant_id}/cross_relations/*.yaml          │
│  或 integration/relations/templates/（平台模板）                    │
├─────────────────────────────────────────────────────────────────┤
│  编排真源（非 relation 文件）                                     │
│  frozen Graph · RuleSet · Skill Pack（Implementation Package）    │
├─────────────────────────────────────────────────────────────────┤
│  差异补丁                                                         │
│  overrides.yaml · OverrideDocument（厂/线/设备级）                  │
└─────────────────────────────────────────────────────────────────┘
```

与 ADR-007 §6 **两层 Connector 模型** 一一对应：A = Definition，B = Instance。

## 3. 配置枢纽目录（落位）

```text
integration/
├── catalog/                              # Layer A：Silver/Gold 认证 Blueprint
│   └── {system}/{vendor}/
│       ├── blueprint.yaml
│       ├── mapping.yaml                  # 可选
│       └── tests/
├── packs/                                # Pack bundle 引用
├── tenants/
│   └── {tenant_id}/                      # 一个 tenant = 一个部署项目单元
│       ├── tenant.yaml                   # 准入、path、shadow、placement
│       ├── system_relations/             # Layer B：「本项目 ↔ 外部系统」
│       │   ├── erp-kingdee.yaml          # 见名知意：system-vendor
│       │   └── im-dingtalk.yaml
│       ├── cross_relations/              # Layer C：「本项目 ↔ 其他项目/租户」
│       │   └── hasen-subsidiary-b.yaml   # 可选；D2+ 集团多厂
│       ├── overrides.yaml                # Override 索引或内联草案
│       └── packs/                        # 已 import 的 Pack 引用清单
├── relations/                            # 平台级关系模板（可选复用）
│   └── path-a-erp-im-work-report.yaml
└── tools/connector-agent/
```

**命名规则**（扩展 [命名约定](./命名约定.md) §4）：

| 路径模式 | 含义 |
|----------|------|
| `system_relations/{system}-{vendor}.yaml` | 单租户绑定单 Legacy 系统（主文件形态） |
| `cross_relations/{peer_tenant_id}.yaml` | 与对端租户协作关系 |
| `tenant.yaml` | 租户元数据；**不含密钥** |
| `relations/templates/*.yaml` | 可复制到新 tenant 的骨架 |

## 4. SystemRelation（系统关系）

### 4.1 职责边界

**SystemRelation 描述**：

- 本 tenant 使用哪个 **Connector Pack / Blueprint**
- 环境（prod/staging）、`base_url`、Edge Agent 绑定
- 凭证 **引用**（`secrets_ref`），非明文
- 指向 catalog 的 **mapping 覆盖**（diff only）

**SystemRelation 不描述**：

- 业务节点顺序、分支、确认门 → **frozen Graph**
- 谁有权写、阈值 → **RuleSet** + **Override**
- HTTP 动词定义 → **catalog/blueprint.yaml**（CMV ops）

### 4.2 文件示例

`integration/tenants/hasen/system_relations/erp-kingdee.yaml`：

```yaml
apiVersion: factoryos.io/v1
kind: SystemRelation
metadata:
  relation_id: sr-hasen-erp-kingdee-prod
  tenant_id: hasen
  name: 哈森灯塔 ERP 生产写读
  path: A                              # ADR-005 Path A/B/C
spec:
  connector:
    pack_id: conn-erp-kingdee-write
    registry_key: erp/kingdee/write
    blueprint_ref: catalog/erp/kingdee-write/blueprint.yaml
    certification_min: silver            # 对外只承诺 Silver+
  environment: prod
  instance:
  # activate 后写入 connector_instances.instance_id
    display_name: 金蝶云星空-哈森主账
    base_url: https://erp.hasen.internal/k3cloud
    secrets_ref: vault://hasen/prod/erp-kingdee   # 不进 Git
    edge_agent_id: edge-hasen-01                  # 私网必填
  mapping_overrides:                   # 仅 diff，不全量复制 catalog
    ops.WORK_REPORT.mapping.work_order_id: $.payload.mo_no
  resilience_overrides:                # 可选；默认继承 blueprint
    timeout_ms: 12000
  shadow:
    inherit_tenant_default: true       # 跟随 tenant.shadow_mode
status:
  lifecycle: draft                     # draft | active | archived
  approved_by: null
  approved_at: null
```

`integration/tenants/hasen/system_relations/im-dingtalk.yaml`：

```yaml
apiVersion: factoryos.io/v1
kind: SystemRelation
metadata:
  relation_id: sr-hasen-im-dingtalk
  tenant_id: hasen
  name: 钉钉 IM 入口
spec:
  connector:
    pack_id: conn-dingtalk
    blueprint_ref: catalog/oa/dingtalk/blueprint.yaml
  environment: prod
  instance:
    secrets_ref: vault://hasen/prod/dingtalk
status:
  lifecycle: active
```

### 4.3 与数据库 `connector_instances` 的映射

| YAML 字段 | DB 列（W1 预埋，S1 启用） |
|-----------|---------------------------|
| `metadata.relation_id` | 业务主键 / 外部 ID |
| `metadata.tenant_id` | `tenant_id` |
| `spec.connector.pack_id` | `pack_id` |
| `spec.environment` | `environment` |
| `spec.instance.base_url` | `base_url` |
| `spec.instance.secrets_ref` | `secrets_ref`（加密后存储） |
| `spec.instance.edge_agent_id` | `edge_agent_id` |
| `status.lifecycle=active` | `status=active` |

**激活流程**（W2+ CLI / Studio Connect 步）：

```text
1. 校验 blueprint_ref 存在且 CMV contract test 通过
2. 合并 catalog mapping + mapping_overrides
3. 写入/更新 connector_instances
4. Audit：integration.relation_activated
```

## 5. CrossRelation（交叉关系）

### 5.1 何时需要

| 场景 | 示例 | 是否需要 |
|------|------|----------|
| 集团母厂读子厂汇总 | hasen ← subsidiary-b 产量只读 | ✅ |
| 同一 Graph 跨系统编排 | ERP 写 + MES 读 | ❌ 用 **Graph 边** |
| 两家独立工厂无数据共享 | — | ❌ 不建文件 |
| 共享 Silver Pack 定义 | 金蝶 Blueprint | ❌ 用 **catalog/** |

### 5.2 职责边界

CrossRelation **只声明**：

- 对端 `tenant_id` 或外部主体
- 关系类型：`read_federation` | `reporting_rollup` | `shared_master_data`
- 允许的 CMV 动词子集（只读为主）
- 凭证与 Endpoint **引用**（不复制 Blueprint）

**禁止**：在 CrossRelation 中定义 L2 写路径或绕过 Rule Engine。

### 5.3 文件示例

`integration/tenants/subsidiary-b/cross_relations/hasen.yaml`：

```yaml
apiVersion: factoryos.io/v1
kind: CrossRelation
metadata:
  relation_id: xr-subsidiary-b-to-hasen
  tenant_id: subsidiary-b
  peer_tenant_id: hasen
  name: 子公司向集团只读汇报
spec:
  relation_type: reporting_rollup
  direction: outbound                      # subsidiary-b → hasen
  allowed_verbs:                           # 仅 L0 读类 CMV
    - QUERY_WO
    - QUERY_REPORT_HISTORY
  peer_connector_ref: sr-hasen-erp-kingdee-prod   # 引用对端 SystemRelation
  schedule:
    reconciliation_cron: "0 2 * * *"       # 对账/汇总 Job
  governance:
    requires_approval: true                # 双方 admin 签字后 activate
status:
  lifecycle: draft
```

母厂侧可存在对称文件 `integration/tenants/hasen/cross_relations/subsidiary-b.yaml`（`direction: inbound`），**双方 active 后**联邦才生效。

## 7. 与其他真源的分工（防重复）

| 配置类型 | 真源 | 关系文件角色 |
|----------|------|--------------|
| CMV 动词、HTTP 路径 | `catalog/blueprint.yaml` | 仅 `blueprint_ref` 引用 |
| 业务链路、确认门 | frozen Graph | 不进入 relation |
| 授权、阈值 | RuleSet + Override | Override 改 Rule 参数 |
| 厂级字段差异 | `mapping_overrides` 或 Override `kind:connector` | 小 diff |
| 实施快照 | ImplementationPackage | export 时 **嵌入** system_relations 摘要 |
| 密钥 | Vault / Edge | 仅 `secrets_ref` |

**合并优先级**（与 [覆盖配置](../规格说明/覆盖配置.md) 一致）：

```text
Scope Override > Tenant Override > SystemRelation.mapping_overrides > catalog/mapping.yaml > Pack 默认
```

## 8. 迁出与复制（配置包语义）

### 8.1 迁出一个项目（tenant）

```text
迁出单元 =
  integration/tenants/{tenant_id}/     # 完整目录（Git tag 或 copy）
+ POST /v1/packages/export              # Implementation Package
+ 凭证在新环境 re-bind（secrets_ref 重指向）
+ 按 Playbook 重跑 Prove / write_approved（治理不可跳过）
```

**代码改动**：S1 路径 **零改动**；S3 仅当 catalog 无 Silver Pack 时需新 Blueprint。

### 8.2 复制到第二家（S1）

```text
1. cp -r integration/tenants/template-starter-a integration/tenants/new-factory
2. 改 tenant.yaml · system_relations 内 base_url / secrets_ref
3. factoryos pack import --file impl-package-d1.json
4. 按 [人工决策 Playbook](../规格说明/人工决策Playbook.md) G-PROVE → G-WRITE-APPROVE
```

周期预期见 [04-工厂实施手册](../../准备/2026-06-16/04-工厂实施手册.md) §一。

### 8.3 Implementation Package 扩展（P1 契约）

当前 [ImplementationPackage.schema.json](../数据结构/ImplementationPackage.schema.json) 含 `connector_configs[]`。  
与 SystemRelation 对齐字段：

| Package 字段 | SystemRelation 字段 |
|--------------|---------------------|
| `pack_id` | `spec.connector.pack_id` |
| `registry_key` | `spec.connector.registry_key` |
| `blueprint_ref` | `spec.connector.blueprint_ref` |
| `secrets_ref` | `spec.instance.secrets_ref` |
| `config` | `mapping_overrides` + `resilience_overrides` 合并快照 |

**W2 任务**：export 时从 active SystemRelation 生成 `connector_configs`；import 时还原为 `system_relations/*.yaml` 草案（`lifecycle=draft`）。

## 9. 校验与 CI

| 检查 | 时机 | 工具（规划） |
|------|------|--------------|
| YAML Schema 合法 | PR | `factoryos relation validate` |
| `blueprint_ref` 可解析 | PR | catalog 存在性 |
| 无环 cross_relation | PR | DAG 检测 |
| `secrets_ref` 无明文 | PR | grep 门禁 |
| contract test | merge 前 | `integration/catalog/.../tests/` |
| activate 后 DB 一致 | 运行时 | connector_instances 对账 Job |

**禁止**：

- 在 `system_relations/` 写 Python 或 Jinja 业务逻辑  
- 在 relation 中设 `shadow_mode=false` 跳过 Prove（须走 [Playbook](../规格说明/人工决策Playbook.md) G-WRITE-APPROVE）  
- `integration/` import `os_core` 非公开 API（[os_core-public-api](./os_core-public-api.md)）

## 10. 与 Studio 六步映射

| Studio 步 | 主要写入配置枢纽 |
|-----------|------------------|
| connect | `system_relations/*.yaml` draft + `tenant.yaml` |
| discover | 无新文件；更新 relation 内 `blueprint_ref` 候选 |
| map | `mapping_overrides` 或 `overrides.yaml` |
| prove | `tenant.shadow_mode`；Audit `integration.write_approved` 前置 |
| freeze | Pack / Graph（非 relation） |
| export | Implementation Package + tenant 目录快照 |
