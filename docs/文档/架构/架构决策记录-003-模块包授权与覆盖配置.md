# 架构决策记录 003：模块包、授权与覆盖配置原则

| 字段 | 值 |
|------|-----|
| 状态 | **Accepted（底座 Phase 0）** |
| 日期 | 2026-06-16 |
| 范围 | 商业化模块边界；与 Platform-L0 内核隔离 |
| 依赖 | ADR-001, ADR-002 |

## 背景

FactoryOS 按 Graph / Capability / Connector **Pack** 模块化授权（05）。D2 定制须 **≥70% Pack 化**（08）。若无正式 Pack/License/Override 架构，6–12 个月内易出现：

- 销售承诺无法映射到代码模块
- 定制逻辑写入 `execution_service` 内核（Whale Tax）
- 第二家工厂无法 import Implementation Package

## 决策

### 1. Pack 是唯一商业—技术边界

- 对外 SKU、License 校验、CAPABILITY-PACK-MAP 追溯 **统一使用 Pack ID**
- 新能力 **先登记 Pack 再实现**；禁止「无 Pack 的功能开关」
- Pack 元数据见 [模块包规格](../规格说明/模块包.md)

### 2. License 硬拒绝

- 未授权 Pack → HTTP **403** + Audit（ADR-002 R-10）
- Phase 1：静态 License JSON；Phase 2：`license-service` DB
- License 校验点：Agent 加载、Rule allow、Execution、Connector Registry

### 3. Override 是唯一的「非 Pack 定制通道**

- 厂/线/设备差异 **仅** 通过 [覆盖配置规格](../规格说明/覆盖配置.md) 配置
- **禁止** 为单客户修改 Platform-L0 源码
- 若 Override 无法表达 → 新 Graph 版本 freeze 或新 Pack；仍不行才评估 L3 项目（≤30%）

### 4. Implementation Package 是复制单元

- 每家 D1/D2 结案 **必须** export Package（08 R4）
- Package 含：frozen Graph 引用、Rule、Licensed Pack 列表、Override、Connector profile（无密钥）
- 第二家 **import 优先于** 重新实施

### 5. 收入层 L0–L3 与技术分层对应

| 收入层 | 技术载体 |
|--------|----------|
| L0 平台订阅 | Platform-L0 + tenant + audit + reconciliation |
| L1 D1 通病 | graph-d1 + cap-report-l2 + conn-mes/erp |
| L2 扩展 Pack | skill-* / graph-* / conn-* (Phase 2+) |
| L3 Override/定制 | OverrideDocument + 实施人天 |

### 6. Coding 预埋（Phase 0 起）

全表含 `tenant_id`；ExecutionRecord 含 `scope_id`（line/equipment，可选）；`license-service` 包占位；禁止 `if (customerId === 'xxx')` 业务分支。

## 不决策（留给 Phase 2+）

- 具体定价数字
- 按写次数计量计费实现细节
- 伙伴 Pack 签名算法

## 后果

- 正面：商业 SKU 与代码边界 1:1；D2 有出口；第二家可复制
- 负面：Phase 1 需维护 Pack JSON；Override 引擎 Phase 2 才有完整能力
