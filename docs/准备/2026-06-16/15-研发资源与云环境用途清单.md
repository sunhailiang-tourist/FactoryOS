# 研发资源与云环境用途清单

> 版本：**v1.0.0** | 日期：2026-06-16  
> 用途：**老板备查**——云用来干什么、测试与生产怎么分、研发工具链花在哪、全流程资源对应  
> 关联：`10` 云采购 · `12` 成本口径 · `14` 执行顺序 · `11` PPT 附录 E/F

---

## 一、一句话总览

```text
阿里云（~3.7～4.6k/月）= 跑 FactoryOS 的「房子」：测试域放代码与验收，生产域只跑业务
Cursor + API（~1～3k/月）= 造房子的「工具」：写代码、联调 LLM/语音识图，不进生产账单混算
人力（另计）= 盖房子的人：内核 → 哈森接入 → 复制
```

**两本账，勿混**（与 `12` 一致）：

| 账本 | 含什么 | 不含什么 |
|------|--------|----------|
| **软硬件合计** | 云 + Cursor + 研发期 LLM/多模态 API + 域名 SSL | 研发工资、销售费用 |
| **云基础设施** | ECS / RDS / Redis / SLB / OSS | Cursor、LiteLLM 账单 |

## 二、环境分离原则

| 维度 | **测试域** | **生产域** |
|------|------------|------------|
| **目的** | 写代码、CI、staging 验收、UAT 预演 | 真实工人/主管使用、哈森 Shadow、多 tenant |
| **谁访问** | 研发 VPN / 办公网白名单 | 钉钉 H5、SLB 公网 HTTPS |
| **数据库** | RDS 测试 PG16（可重建） | RDS 生产 PG16（备份 15 天） |
| **绝对禁止** | — | GitLab、YApi、在 ECS 上自建 PG |

## 三、测试域：每一块云资源干什么

> 拓扑见 `10` §二；本节回答「采购回来具体跑什么」。

### 3.1 测试 ECS（4C16G + 150G 数据盘）

| 部署项 | 软件 / 组件 | 用途 | 对应研发阶段 |
|--------|-------------|------|--------------|
| **① 代码仓库** | GitLab CE（Omnibus） | 源码、MR、版本标签、发布触发 | W1 起全程 |
| **② CI 流水线** | GitLab Runner（concurrent 1～2） | `pytest`、契约测试（AC-BASE-001）、构建镜像/rsync | 每次 push |
| **③ 测试系统部署** | FactoryOS **staging** | 与生产同 Compose 结构；接 RDS 测试 | Gate 0 联调、UX-001 手测 |
| **④ 接口文档展示** | Nginx + **Redoc 静态页** | 展示 `文档/接口/工厂操作系统-v1.1.yaml` | 对内 API 查阅 |
| **⑤ 域名入口** | `gitlab.*` + 通配符 SSL | 研发 HTTPS 访问 | 采购日配置 |

**明确不部署**：

- **YApi**（少维护一套 Mongo；OpenAPI **published** 在 Contract Registry，Git/`contracts/` 为 export 镜像）
- 生产级 Redis（staging 可 mock 或轻量连接，对账 Job 以 stub 为主）

### 3.2 RDS 测试（PG16 2C4G 50G）

| 存什么 | 不存什么 |
|--------|----------|
| staging 的 Graph / Rule / Audit / ExecutionRecord | 哈森真实业务全量 |
| UAT 演练数据、mock tenant | 客户 Layer 3 数仓 |
| 迁移脚本验证 | 生产备份混用 |

### 3.3 测试域典型日流程

```text
研发本地（Cursor）改代码
  → git push GitLab（测试 ECS）
  → Runner：pytest + contract（mock LLM / mock 多模态，$0）
  → 部署 staging（RDS 测试）
  → 人手联调：可走真 LiteLLM + 真 ASR/OCR（计入研发 API，见 §五）
  → 通过后 tag / 人工批准 → 发布生产（§四）
```

## 四、生产域：每一块云资源干什么

### 4.1 SLB + ECS 生产（4C16G）

| 部署项 | 用途 |
|--------|------|
| **SLB** | 公网 HTTPS 入口；`api.*` 域名；健康检查 |
| **Nginx** | 反向代理（TLS 可在 SLB 终结） |
| **FactoryOS API** | FastAPI + Uvicorn ×4；**仅运行时**，无 GitLab |

**租户**：哈森 + 第 2 家 = 同一栈上不同 `tenant_id`（逻辑隔离）。

### 4.2 RDS 生产（PG16 4C8G 100G）

| 数据层 | 内容 | 文档 |
|--------|------|------|
| **Data-L1** | Graph/Rule 版本、Audit、ExecutionRecord、对账日志、License | `06` §四 |
| **Data-L2** | 最小业务缓存（TTL）；能读 ERP 就不落库 | `06` §四 |
| **不含** | 客户 ERP 全量副本（Layer 3 选配另立项） | `06` |

### 4.3 Redis 1G + OSS 100G

| 资源 | 用途 | 阶段 |
|------|------|------|
| **Redis** | 会话、限流、缓存；Phase 2 对账 Job 队列 | Day1 接通接口 |
| **OSS** | 语音/图片（感知层）；**不进 PG**；30 天生命周期 | 多模态入口 |

### 4.4 生产出站（安全组放行）

| 目标 | 用途 |
|------|------|
| 客户 **ERP** API / 网段 | `conn-erp` 读+写（哈森报工） |
| **钉钉** 开放平台 | H5、待办、通知 |
| **LiteLLM** Provider | 运行时意图/填槽（试点后按 tenant 预算） |
| **ASR/OCR/VLM** | 工人真实语音/识图（试点后按量，与研发期分开记账） |

## 五、非云研发资源：Cursor 与 API

> 明细与控费见 `12` §二～§三、§六。

### 5.1 Cursor（研发工具 · 不走 LiteLLM）

| 项 | 说明 |
|----|------|
| **是什么** | 研发人员写 FactoryOS 代码的 AI IDE（Pro+ $60/人/月） |
| **用来干什么** | 内核 package 编码、AC 驱动补全、重构、生成测试 |
| **跑在哪** | **研发本机**；不部署在阿里云 ECS 上 |
| **与产品的关系** | 客户工人 **不用** Cursor；仅内部研发 |

### 5.2 应用 LLM（LiteLLM · 产品研发 API）

| 场景 | 环境 | 谁付钱（账本） |
|------|------|----------------|
| 单元测试 / CI | mock | **¥0** |
| staging 手测、意图填槽联调 | 测试 ECS staging | **研发 API**（`12`） |
| Shadow / 真实工人 | 生产 ECS | **tenant 试点预算**（与研发分开） |

**典型模型**：DeepSeek-V3 / 通义 Qwen-Turbo（开发）；staging 可略升档。

### 5.3 多模态 API（ASR / OCR / VLM）

| 能力 | 开发期用途 | 记账 |
|------|------------|------|
| **ASR** | UX-001 语音报工联调 | 研发 API |
| **OCR** | 工单拍照识别 fallback | 研发 API |
| **VLM** | 少量 staging 手测 | 研发 API |
| **生产流量** | 工人真实说/拍 | OSS 存媒体 + API 按量（tenant） |

**控费铁律**：CI 强制 mock；优先扫码 fallback（`D-03`）。

## 七、内核如何支撑制造业核心系统「优雅扩展」

> 配图：系统架构图 · 技术架构图 · 数据架构图（`文档/架构/`）

### 7.1 不变的内核（Platform-L0）

```text
Graph（frozen）→ Rule → DSL → Execution（唯一写路径）→ Audit + Revert
```

**所有行业、所有 Legacy 系统** 都走同一条写链；扩展 **不改内核**。

### 7.2 扩展靠 Connector Pack（Platform-L1）

| 制造业系统 | system 枚举 | 典型能力 | 接入方式 |
|------------|-------------|----------|----------|
| **ERP** | `erp` | 订单、库存、**报工写（哈森）** | `conn-erp-{vendor}-read/write` |
| **MES** | `mes` | 工序、现场报工写 | `conn-mes-{vendor}-write` |
| **WMS** | `wms` | 发料、库位 | Phase 2 stub → Pack |
| **QMS** | `qms` | 检验、NCR | 只读 Pack |
| **APS** | `aps` | 排产 | 建议只读 |
| **PLM** | `plm` | BOM/工艺 | 只读引用 |
| **CRM/CSM** | `crm`/`csm` | 订单/售后 | Phase 2+ |
| **OA/IM** | `oa` + 钉钉 Pack | 待办、确认 | `conn-dingtalk` |
| **无系统** | `mes` builtin | 内置账本 | `conn-mes-builtin`（B-Lite） |

**扩展纪律**（`连接器` §3.2）：

1. 新厂商 = 新 Pack，**不**新增 `system` 类型  
2. 未注册 Connector → 403，不 silent fail  
3. 新系统进 Graph = **clone 新版本 + freeze**，不改旧图  
4. **禁止** `if (system === 'qms')` 硬编码；统一 Registry  

### 7.3 双切入路径（图上看两条线，内核一条）

| 路径 | 客户 | Connector 组合 |
|------|------|----------------|
| **A · Overlay** | 哈森（有 ERP + 钉钉） | ERP 读+写 + 钉钉 |
| **B · 通用 MES 厂** | 有 MES 的客户 | MES 写 + ERP 读 |
| **C · B-Lite** | 无 ERP/MES | 内置账本 Connector |

三条线 **同一 OS 内核**；仅 Registry 里装的 Pack 不同。

## 九、采购与就绪检查清单

```text
□ 测试 ECS：GitLab + Runner + staging + Redoc
□ 生产 ECS：仅 Nginx + API（无 GitLab）
□ RDS ×2（测试/生产）拓扑对称
□ Redis + OSS + SLB + 通配符 SSL
□ 域名：api.* → SLB；gitlab.* → 测试 EIP
□ Cursor Pro+ × N，设 spend limit
□ LiteLLM：dev/staging 默认便宜模型；CI mock
□ 多模态测试账号（ASR/OCR）；CI mock
□ ERP 出站：VPN 或客户白名单（哈森）
□ 钉钉企业应用审批
```
