## 一、执行顺序（拍板后行为链）

```text
① PPT 汇报 → 商业/架构/技术/成本 → 老板确认方向与资源原则
② 内核冲刺（6～8 周）→ Gate 0：AC-BASE-001 全绿
③ 云采购（与②并行）→ 按 `10` 定版上阿里云
④ 哈森接入（内核绿后）→ ERP Connector + 钉钉 H5 → D1 报工闭环
⑤ 按需加人 → 实施 Shadow/UAT；成熟后再开 B-Lite / 第二家 / MES 厂
```

**不变**：Overlay、D1 五项、写路径红线、Gate 0 / Gate 0' 分界。

---

## 二、哈森灯塔切入策略（整体调整 ★）

> **不是 MES 优先**；哈森现状：**ERP 为主账本与业务落点，OA = 钉钉**。

| 层 | 哈森首期 | 说明 |
|----|----------|------|
| **写路径（报工）** | **`conn-erp-{vendor}-write`**（或 ERP 生产/报工 API） | 无独立 MES；产量进 ERP |
| **读路径** | **`conn-erp-{vendor}-read`** | 工单、订单、库存查询 |
| **入口 / 确认** | **`conn-dingtalk`** | H5、待办、主管确认、通知 |
| **MES** | **不接**（灯塔期） | Registry 预留；有别厂或哈森上 MES 时再 `conn-mes-*` |

**价值场景（不变）**：工人钉钉报工 → 主管确认 → **ERP 有数** → 可 Revert、可对账。

**与平台能力关系**：FactoryOS **仍支持** MES 写路径（其他客户）；哈森 lighthouse **只绑 ERP + 钉钉**。

## 三、阶段与人力（内核优先，非 Day1 双轨）

### 3.1 阶段表

| 阶段 | 时间 | 研发 | 实施/业务 | 产出 |
|------|------|------|-----------|------|
| **P0 内核** | W1～W8 | **1～2 后端** | 0.5 人收 ERP API 文档 | Gate 0；mock Connector 即可 |
| **P1 哈森接入** | W7～W12 | +1 后端 ERP + 1 前端/H5 | **1 人** Graph/摸底 | 真实 ERP + 钉钉闭环 |
| **P2 哈森 D1** | W10～W22 | 维护 | **1 人** Shadow/UAT | D1 书面结案（≤90 天窗） |
| **P3 扩线** | W16+ 按需 | +1 人 | 按需 | B-Lite / 第 2 家 / 有 MES 厂 |

### 3.2 12 个月目标（初具规模）

| 指标 | 目标 |
|------|------|
| 内核 + Pack 机制 | M1～M2 完成 |
| 哈森 **≥1 实体** D1 结案 | M4～M5 |
| 第 2 家或第二实体 import | M6～M8 |
| 累计 **3～5 家** tenant | M12 |
| 无系统厂 B-Lite | **P1 验证后** 再加 1 研发，不抢哈森主线 |

### 3.3 人力建议

| 配置 | 人数 | 适用 |
|------|------|------|
| 最低 | **3 人**（2 研发 + 1 实施） | 内核 + 哈森 1 家 D1 |
| **推荐** | **4～5 人** | 哈森 + 并行 B-Lite/复制 |
| 拍板后首月 | **2 研发 + 云采购** | 仅 P0，不接厂 |
